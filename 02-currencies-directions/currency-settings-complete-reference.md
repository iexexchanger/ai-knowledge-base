# Валюты: полный порядок настроек

Эта страница описывает карточку валюты в том порядке, в котором администратор видит разделы в меню. Валюта в iEXExchanger — это не только код `USDT` или `BTC`. Это полноценная настройка способа приема или выплаты: платежная система, сеть, поля реквизитов, инструкции, верификация, мерчанты, автовыплаты, AML, резерв, реквизиты, пересчет и дополнительные опции.

## С чего начинать новичку

1. Создайте или проверьте платежную систему: название, логотип, понятное отображение.
2. Создайте или проверьте код валюты: `USDT`, `BTC`, `RUB`, `USD`.
3. Создайте валюту из пары "платежная система + код валюты".
4. Откройте карточку валюты и заполните вкладки по порядку.
5. Настройте резерв.
6. Настройте реквизиты, мерчант или автовыплату.
7. Создайте направление обмена.
8. Проверьте публичную форму и тестовую заявку.

## 1. Основное

| Поле | Как работает | Как заполнять | Пример |
|---|---|---|---|
| Статус | Определяет, можно ли использовать валюту | Активная для рабочих валют, неактивная для скрытия, архивная для старых | Активная |
| Платежная система | Дает логотип и общий тип способа оплаты | Выбирайте правильную систему, не смешивайте разные сети | Tether, Bitcoin, Сбербанк |
| Код валюты | Денежный код для расчетов | Используйте стандартные коды | USDT, BTC, RUB |
| Отображать код валюты | Показывает код рядом с названием | Включайте для крипты и похожих валют | `Tether USDT TRC20` |
| Техническое название | Название в админке и на сайте | Пишите так, чтобы клиент понял сеть и способ | `Tether USDT TRC20` |
| XML-обозначение | Код для XML/мониторингов | Выберите подходящий XML-код или задайте вручную | `USDTTRC20` |
| Формат числа | Количество знаков после запятой | Для BTC больше, для фиата меньше | BTC: 8, RUB: 2 |
| Конвертировать по | Код для пересчета и аналитики | Обычно совпадает с базовым кодом | USDT конвертировать по USDT |
| Название по языкам | Человекочитаемый заголовок | Заполните основной язык минимум | `USDT TRC20` |

### Пример USDT TRC20

| Поле | Значение |
|---|---|
| Платежная система | Tether |
| Код валюты | USDT |
| Техническое название | Tether USDT TRC20 |
| XML | USDTTRC20 или код мониторинга |
| Формат числа | 2-6 по регламенту обменника |
| Показывать код | Да |

## 2. Поля

Эта вкладка управляет тем, какие данные клиент вводит при создании заявки.

| Поле | Как работает | Рекомендация |
|---|---|---|
| Доп. поля "Отдаю" | Добавляет дополнительные поля для стороны, которую клиент отдает | Используйте только нужные поля: банк, ФИО, комментарий |
| Доп. поля "Получаю" | Добавляет дополнительные поля для стороны получения | Для банковской выплаты часто нужны банк/ФИО/карта |
| Название главного поля | Как называется поле счета/кошелька | "Адрес USDT TRC20", "Номер карты", "IBAN" |
| Комментарий главного поля | Подсказка под полем | Объясните формат и сеть |
| Скрыть поле "Отдаю" | Не показывать реквизит на стороне отдачи | Включайте только если реквизит не нужен |
| Скрыть поле "Получаю" | Не показывать реквизит на стороне получения | Для выплат обычно не включать |
| Первый символ | Подсказка/начальный символ реквизита | Используйте только для стабильных форматов |
| Min/Max символов | Ограничение длины реквизита | Подберите реальную длину, иначе заявка не создастся |
| Текст ошибки min/max | Что увидит клиент при неверной длине | Пишите простым языком |
| Название поля "Отдаю/Получаю" | Разные подписи для разных сторон | Полезно, если отдача и получение отличаются |
| Комментарий "Отдаю/Получаю" | Подсказка по сторонам | Для крипты обязательно укажите сеть |
| Маска | Формат ввода | Карта: `0000 0000 0000 0000`; IBAN/SWIFT по формату |
| Placeholder маски | Символ-заполнитель | Не делайте его слишком заметным |
| Допустимые символы | Любые, числа, буквы, латиница, латиница+цифры | Карты: числа; crypto: латиница+цифры |
| Удалять пробелы | Автоматически чистит пробелы | Включайте для карт и кошельков |
| Валидатор "Отдаю/Получаю" | Проверяет реквизит по правилу | Выбирайте валидатор под валюту и сеть |
| Ошибка валидатора | Текст ошибки для клиента | "Проверьте адрес и сеть TRC20" |
| Сканировать QR | Разрешает QR-сканирование | Удобно для кошельков |

### Пример для банковской карты

| Поле | Значение |
|---|---|
| Название поля | Номер карты |
| Маска | `0000 0000 0000 0000` |
| Допустимые символы | Только числа |
| Удалять пробелы | Да |
| Min/Max | 16-18 по правилам банка |

### Пример для crypto-кошелька

| Поле | Значение |
|---|---|
| Название поля | Адрес USDT TRC20 |
| Комментарий | Укажите адрес только в сети TRC20 |
| Маска | Обычно пусто |
| Допустимые символы | Латиница и цифры |
| Валидатор | Валидатор адреса нужной сети |

| Описание обмена | Краткое описание валюты | Для пояснений на форме |
| Источник описания | Авто, направление, валюта | Если направление имеет отдельные условия, выбирайте направление |
| Формализация | Дополнительный текст оформления | Когда нужна юридическая или операционная фраза |
| Документы "Отдаю" | Документы при оплате | Только если реально нужны |
| Документы "Получаю" | Документы при выплате | Согласуйте с KYC |

## 4. Верификация карты

| Поле | Как работает | Пример настройки |
|---|---|---|
| Условия проверки | Выключено, всегда или от суммы | Для банковских карт часто "от суммы" |
| Минимальная сумма | Порог, с которого нужна проверка | `50000` RUB, если таков регламент |
| Текст проверки | Краткое объяснение клиенту | "Для безопасности подтвердите карту" |
| Информация | Подробная инструкция | Какое фото нужно, что закрывать нельзя |
| Разрешить в кабинете | Можно ли проходить проверку из личного кабинета | Включайте, если клиентам нужен повторный доступ |

## 5. Верификация личности

| Поле | Как работает | Важно |
|---|---|---|
| Режим KYC | Выключено, всегда, от суммы | Не включайте без понятной инструкции |
| Минимальная сумма | Порог KYC | Должен совпадать с политикой риска |
| Поведение непроверенного клиента | Блокировать сразу или разрешить и попросить проверку | Для строгих валют лучше блокировать |
| Текст | Краткое пояснение | Клиент должен понять, зачем KYC |
| Информация | Подробная инструкция | Укажите сроки и следующий шаг |

## 6. Мерчанты

Мерчанты используются на стороне приема оплаты. Если валюта стоит в направлении как "Отдаю", выбранный мерчант может выдавать реквизиты или подтверждать платеж.

| Поле | Как работает | Важно |
|---|---|---|
| Список мерчантов | Какие провайдеры можно использовать | Выбирайте только проверенные |
| Общая сеть | Сеть, которую передавать по умолчанию | Например `TRC20` |
| Сеть по мерчанту | Индивидуальный код сети для конкретного провайдера | Нужен, если API ждет другой код |
| Валидатор мерчанта | Проверка реквизита, полученного от мерчанта | Защищает от ошибочных карт/адресов |
| Стратегия ошибки валидатора | Игнорировать или остановить | Для денег лучше останавливать |
| Лимит суммы за день/месяц | Сколько можно провести через мерчант | Ставьте по договору и риску |
| Min/Max суммы заявки | Лимит одной заявки | Ниже/выше лимита мерчант не должен использоваться |
| Лимит заявок за день/месяц | Количество операций | Полезно при лимитах API |

## 7. Выплаты

Выплаты используются, когда валюта стоит в направлении как "Получаю" и обменник должен отправить клиенту деньги.

| Поле | Как работает | Важно |
|---|---|---|
| Список выплат | Провайдеры автовыплаты | Перед включением делайте тест |
| Общая сеть выплаты | Сеть по умолчанию | Должна совпадать с валютой |
| Сеть по выплате | Индивидуальный код сети провайдера | Ошибка может привести к отказу или потере средств |
| Разрешить автовыплату | По умолчанию, нет, да | На старте лучше включать осторожно |
| Лимит суммы за день/месяц | Ограничение объема выплат | Защищает баланс |
| Min/Max суммы заявки | Лимит одной выплаты | Ставьте по провайдеру |
| Лимит заявок за день/месяц | Ограничение количества выплат | Полезно против перегрузки |

## 8. AML

AML по валюте управляет проверкой кошелька или транзакции.

| Поле | Как работает | Пример |
|---|---|---|
| Источник AML | Какой сервис выполняет проверку | GetBlock, Chain сервис и т.п. |
| Проверка кошелька | Нет, перед созданием заявки, при проверке мерчанта | Для crypto-входа можно проверять до заявки |
| Сумма "от" для кошелька | Порог запуска проверки | `0` — без ограничения, если так работает настройка |
| Если риск превышен | Ничего не делать или показать ошибку | Строгая политика: показать ошибку |
| Проверка TX | Нет, при оплате, при автовыплате | TX точнее адреса |
| Сумма "от" для TX | Порог проверки транзакции | Используйте по регламенту |
| AML текст "Отдаю" | Текст для стороны оплаты | Объяснить проверку риска |
| AML текст "Получаю" | Текст для стороны выплаты | Предупредить о возможной проверке |

## 9. Лимиты резерва

| Поле | Как работает | Как использовать |
|---|---|---|
| Максимальный лимит в резерве | Верхний предел внутреннего резерва | Не ставьте выше реальных средств |
| Максимальный отображаемый резерв | Сколько показывать клиенту | Можно показывать меньше фактического |
| Процент прибыли резерва | Процентная корректировка резерва/прибыли | Используйте только если понимаете схему |
| Заявки "ожидает" в час | Ограничение заявок в ожидании | `0` обычно отключает ограничение |
| Заявки "в обработке" в час | Ограничение заявок в процессе | Защищает от перегрузки оператора |
| Дневной лимит "Отдаю" | Сколько можно принять по валюте за день | Согласуйте с резервом и лимитами банка/кошелька |
| Дневной лимит "Получаю" | Сколько можно выплатить за день | Важно для автовыплат |
| Месячный лимит вход/выход | Ограничение объема за месяц | Для финансового контроля |
| Процент переноса резерва | Перенос части резерва | Осторожно со связанными резервами |
| Сумма переноса | Фиксированный перенос | Не используйте без теста |

## 10. Реквизиты

| Поле | Как работает | Когда использовать |
|---|---|---|
| Метод выдачи реквизитов | Стандартно или по запросу | По запросу, если реквизиты выдает оператор |
| Тип вывода реквизитов | По умолчанию, случайно, по порядку/случайно | Для ручных реквизитов выбирают схему распределения |
| Текст запроса реквизитов | Что видит клиент, пока ждет реквизиты | Заполняйте при режиме "по запросу" |

### Когда выбирать "по запросу"

Используйте режим "по запросу", если реквизиты нельзя показывать автоматически: например, оператор должен проверить сумму, банк клиента, лимит карты или доступность конкретного счета.

## 11. Пересчет

Пересчет валюты управляет заявками, где сумма может меняться из-за курса.

| Поле | Как работает | Рекомендация |
|---|---|---|
| Включить пересчет | Разрешает уникальный пересчет заявок | Не включайте без текста для клиента |
| Статусы пересчета | В каких статусах можно пересчитывать | Не выбирайте финальные статусы |
| Режим | Нет, всегда, по правилам | "По правилам" безопаснее |
| Процент изменения курса | Пересчитывать при изменении от процента | Например 0.5 или 1 |
| Интервал минут | Пересчитывать через время | Используйте вместе с порогом или отдельно |
| Только если курс изменился | Не пересчитывать без изменения курса | Обычно включать |
| Мин. сумма "Отдаю" | Не пересчитывать мелкие заявки | Полезно для низких сумм |
| Направление курса | Любое, вверх, вниз | Используйте, если важен только рост/падение |
| Порог вверх/вниз | Отдельные пороги изменения | Для защиты от мелкого шума |
| Тихое окно | Пауза после создания/изменения | Чтобы клиент успел оплатить |
| Anti-storm | Защита от частого пересчета | Чтобы заявка не менялась слишком часто |
| Максимум пересчетов | Сколько раз можно пересчитать в статусе | `0` обычно без лимита |
| Максимальный возраст в статусе | После какого времени не пересчитывать | Защита старых заявок |
| Текст изменения курса | Сообщение клиенту | Пишите простым языком |

## 12. Партнер

Эта вкладка относится к партнерским выплатам по валюте, а не к общей партнерской программе.

| Поле | Как работает | Важно |
|---|---|---|
| Партнер по умолчанию | Использовать стандартную партнерскую логику | Включайте, если валюта участвует в общей схеме |
| Индивидуальный партнер | Разрешить индивидуальные условия | Используйте для особых валют |
| Комиссия ПС, % | Процентная комиссия по партнерской выплате | Учитывайте экономику направления |
| Комиссия ПС, S | Фиксированная комиссия | `S` — технический маркер фиксированного значения, не доллар |

## 13. Опции

| Поле | Как работает | Когда включать |
|---|---|---|
| QR-код | Показывает QR для реквизита | Для crypto и карт, если удобно клиентам |
| Prefix QR | Добавляет префикс в QR | Например схема кошелька |
| Сумма в QR | Добавляет сумму в QR | Если кошелек/банк поддерживает |
| Прикрепление файла | Клиент может приложить чек/документ | Для банковских оплат и спорных сценариев |
| Название файла | Текст поля файла | "Прикрепите чек" |
| Описание файла | Инструкция по файлу | Что именно приложить |
| Разрешить заявки | Можно ли создавать заявки по валюте | Отключайте для временной остановки |
| Шаги заявки | Показывать пошаговую проверку | Для сложных сценариев |
| Email verification modal | Просить подтвердить email | Для риска фейковых email |

## Проверка после настройки

| Проверка | Что должно быть |
|---|---|
| Валюта активна | Есть правильный статус и название |
| Поля понятны | Клиент понимает, что вводить |
| Сеть указана | Для crypto сеть видна в названии/подсказке |
| Резерв есть | Резерв соответствует реальным средствам |
| Мерчант/реквизиты работают | Клиент получает реквизиты или заявка идет оператору |
| Выплата проверена | Для автовыплаты есть тест |
| AML/KYC понятны | Есть инструкции и пороги |
| Направление создано | Валюта участвует хотя бы в одной рабочей паре |
| Тестовая заявка прошла | От публичной формы до финального статуса |

## Частые ошибки

| Ошибка | Причина | Что сделать |
|---|---|---|
| Валюта не видна | Неактивна, нет направления, фильтр, кеш | Проверить статус, направление, фильтр, кеш |
| Клиент не может ввести реквизит | Маска, валидатор, min/max, символы | Ослабить правило или исправить текст ошибки |
| Мерчант не выдает реквизиты | Нет мерчанта, сеть, лимит, валидатор | Проверить вкладку "Мерчанты" и журнал |
| Автовыплата не уходит | Нет провайдера выплат, запрет, лимит, сеть | Проверить вкладку "Выплаты" и логи |
| AML блокирует всех | Неточно настроен сервис/порог/ошибка | Проверить AML-вкладку и журнал AML |
| Резерв кажется неправильным | Лимит отображения, файл, связанный резерв | Проверить "Лимиты резерва" и раздел "Резервы" |
| Партнерская комиссия непонятна | Перепутали `%` и `S` | `%` процент, `S` фиксированное значение |
