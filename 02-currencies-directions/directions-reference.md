# Подробный справочник направлений обмена

Направление — это конкретная пара обмена, например `BTC - USDT`. Именно направление объединяет две валюты, курс, лимиты, резерв, комиссии, мерчант, автовыплату, поля, KYC, AML, города, экспорт и правила обработки заявки.

Для вкладок **Основная**, **Доп. и ПС**, **От суммы обмена**, **Пересчет заявок** и **Ограничения и проверки** используйте отдельный подробный справочник: [Комиссии, пересчет и ограничения направления](direction-fees-recount-restrictions-reference.md). Там разобраны стороны "Отдаю/Получаю", минимальные комиссии, диапазоны суммы, FIX/FLOATING, страны, IP, лимиты новичков и лимиты повторных заявок.

## Список направлений

| Поле / действие | Что означает | Как использовать | Ошибки при неверной настройке |
|---|---|---|---|
| Фильтр по валюте “Отдаю” | Показывает направления, где клиент отдаёт выбранную валюту | Используйте для проверки пары | Можно перепутать сторону пары |
| Фильтр по валюте “Получаю” | Показывает направления, где клиент получает выбранную валюту | Используйте для обратной пары | `BTC - USDT` и `USDT - BTC` — разные направления |
| Фильтр по статусу | Активные, выключенные, архивные | Быстро найти скрытые направления | Выключенное направление не видно клиенту |
| Массовое включение/выключение | Меняет статус нескольким направлениям | Используйте при технических работах | Можно случайно выключить рабочие пары |
| Дублировать направление | Создаёт копию пары | Удобно для похожих направлений | После дубля обязательно поменяйте валюты, курс, лимиты |
| Архив | Убирает направление из работы | Используйте для старых пар | Не архивируйте пару с активными заявками |
| Проверка проблем | Показывает причины недоступности | Используйте, если пара не видна | Причины могут быть в валюте, курсе, резерве или лимитах |

## Создание направления

| Поле / настройка | Что означает | Как правильно заполнять | Рекомендация |
|---|---|---|---|
| Валюта “Отдаю” | Что клиент отправляет обменнику | Выберите активную валюту | Сторона важна для реквизитов и мерчанта |
| Валюта “Получаю” | Что клиент получает | Выберите активную валюту | Сторона важна для выплаты и резерва |
| Техническое название | Название направления в админке | Например `BTC - USDT TRC20` | Не используйте непонятные сокращения |
| Статус | Доступность направления | Включайте после настройки курса, резерва и лимитов | Не включайте пустую пару |
| Проверка существующей пары | Предупреждает о дубле | Если пара уже есть, откройте её | Дубли путают сайт и операторов |

## Основные настройки направления

| Поле / настройка | Что означает | Как правильно заполнять | Рекомендация |
|---|---|---|---|
| Статус | Активность пары | Активно, выключено, архив | Активируйте только после теста |
| Валюта 1 | Сторона “Отдаю” | Выберите правильную валюту | После заявок менять сторону опасно |
| Валюта 2 | Сторона “Получаю” | Выберите правильную валюту | Ошибка создаёт неправильные заявки |
| Техническое название | Внутреннее название пары | Используйте понятный формат | Например `BTC - USDT TRC20` |
| Минимум “Отдаю” | Минимальная сумма, которую клиент может отдать | Укажите реальный бизнес-минимум | Слишком высокий минимум снижает заявки |
| Максимум “Отдаю” | Максимальная сумма входа | Укажите по резерву, мерчанту и рискам | Не ставьте выше возможностей оплаты |
| Минимум “Получаю” | Минимальная сумма выплаты | Рассчитывайте по лимитам автовыплат | Может блокировать мелкие заявки |
| Максимум “Получаю” | Максимальная сумма выплаты | По резерву и лимитам провайдера | Не ставьте выше резерва |
| Источник лимитов | Откуда брать min/max | Базовые, профиль, вручную, без лимита | Профили удобны для массового управления |
| Профиль лимитов | Готовый набор лимитов | Выберите подходящий профиль | Изменение профиля влияет на все привязанные направления |
| Сделать главным | Использовать направление как приоритетное | Включайте для популярных пар | Не делайте главным неготовую пару |

## Информация направления

| Поле / настройка | Что означает | Как правильно заполнять | Рекомендация |
|---|---|---|---|
| Описание обмена | Текст о конкретной паре | Опишите особенности `BTC - USDT` | Если правила общие, можно брать описание валюты |
| Дополнительное описание | Второй текстовый блок | Используйте для особых условий | Не дублируйте одно и то же |
| Инструкция оплаты | Что должен сделать клиент | Пошагово, без технического шума | Критично для ручных реквизитов |
| Текст подтверждения | Что показывать перед созданием заявки | Укажите важные предупреждения | Клиент должен понимать последствия |
| Кнопка подтверждения | Текст кнопки | Коротко: “Подтвердить”, “Создать заявку” | Длинный текст плохо выглядит |
| Интервал подтверждения | Задержка или время подтверждения | Укажите только если нужен процесс ожидания | Не используйте без смысла |
| SEO title | Заголовок страницы пары | Коротко по паре | Полезно для публичных страниц направлений |
| SEO description | Описание для поиска | 1–2 предложения | Не вставляйте внутренние данные |
| SEO keywords | Ключевые слова | Умеренно и по смыслу | Не перегружайте |

| Сортировка | Порядок полей | Важные поля выше | Логичный порядок повышает удобство |

## Курсы направления

| Поле / настройка | Что означает | Как правильно заполнять | Рекомендация |
|---|---|---|---|
| Источник курса | Откуда направление берёт курс | Источник, формула, файл, конкурент, BestChange или ручной курс | Выбирайте один понятный источник |
| Добавка к курсу в процентах | Процентная корректировка | Например `1` для плюс 1 процент | Проверяйте итоговый курс на сайте |
| Добавка к курсу фиксированная | Фиксированная надбавка | Используйте редко, когда нужна абсолютная поправка | Для разных сумм может вести себя грубо |
| Ручной курс | Фиксированный курс направления | Укажите только при ручном управлении | При ручном курсе автоисточники не помогают |
| Формула | Формульный источник | Выберите готовую формулу | Формула должна возвращать число |
| Файловый курс | Курс из файла | Выберите пару из файлового источника | Проверьте preview файла |
| Конкурент | Курс конкурента | Выберите источник конкурента | Используйте для мониторинга рынка |
| Минимальная сумма для нового курса | Сумма, от которой применять другой курс | Заполняйте при ступенчатых курсах | Важно для крупных заявок |
| Максимальная сумма для нового курса | Верхний порог | Укажите диапазон | Диапазоны не должны конфликтовать |
| Новый курс при диапазоне | Какой курс применить | Выберите курс или добавку | Проверьте на тестовой сумме |

## Пересчёт направления

| Поле / настройка | Что означает | Как правильно заполнять | Рекомендация |
|---|---|---|---|
| Включить пересчёт | Разрешить пересчитывать заявку | Включайте для плавающих курсов | Нужно объяснить клиенту |
| Статусы пересчёта | На каких статусах можно пересчитать | Выбирайте активные статусы до выплаты | Не пересчитывайте финальные заявки |
| Направление изменения курса | Любое, только вверх, только вниз | Выберите по бизнес-логике | Защищает от нежелательных пересчётов |
| Порог роста | На сколько должен вырасти курс | Укажите процент/значение | Слишком низко создаст частые пересчёты |
| Порог падения | На сколько должен упасть курс | Укажите значение | Согласуйте с клиентскими условиями |
| Минимальная сумма отдачи | Ниже какой суммы не пересчитывать | Укажите для защиты мелких заявок | Не усложняйте малые обмены |
| Тихое окно | Время без пересчёта после создания | Укажите минуты | Защищает от мгновенных изменений |
| Anti-storm | Защита от слишком частых пересчётов | Включайте при волатильных курсах | Без неё заявка может пересчитываться много раз |
| Максимум пересчётов | Сколько раз можно пересчитать | Ограничьте разумно | Иначе клиент не поймёт итог |
| Максимальный возраст статуса | Сколько минут статус может пересчитываться | Укажите по регламенту | Старые заявки лучше обрабатывать вручную |
| Текст пересчёта | Сообщение клиенту | Объясните, почему сумма изменилась | Очень важно для доверия |

Подробная логика FIX/FLOATING, статусов, интервалов и порогов описана в разделе [Комиссии, пересчет и ограничения направления](direction-fees-recount-restrictions-reference.md).

| Заморозить направление | Запретить создание заявок, но оставить пару видимой | Используйте при временной паузе | Для полного скрытия лучше выключить направление |
| Расписание | Рабочее время направления | Укажите время "От" и "До" | Полезно для наличных и ручных направлений |
| Запрещенные IP | IP, которым нельзя создавать заявки | Один IP с новой строки | За proxy сначала настройте реальный IP |
| Запрещенные страны | Страны, из которых заявка запрещена | Выберите страны блокировки | Deny-список имеет приоритет |
| Разрешенные страны | Единственные страны, из которых заявка разрешена | Заполняйте только для строгой географии | Если список заполнен, остальные страны будут закрыты |
| Только для верифицированных | Требовать верифицированный аккаунт | Включайте для рискованных направлений | Клиент без проверки не создаст заявку |
| Успешные обмены | Минимальная история клиента | `0` отключает ограничение | Используйте для VIP или рискованных пар |
| Лимит новичка | Максимальная сумма "Отдаю" для нового клиента | Укажите безопасную сумму | Не ставьте слишком низко без причины |
| Лимиты заявок | Ограничение по IP, пользователю, e-mail и счетам | Заполняйте общий и дневной лимиты | `0` обычно означает отсутствие лимита |

Подробные примеры и сценарии по этой вкладке находятся в разделе [Комиссии, пересчет и ограничения направления](direction-fees-recount-restrictions-reference.md).

## Частые проблемы по направлениям

| Ситуация | Причина | Что сделать |
|---|---|---|
| Пара не видна на сайте | Статус, валюта, фильтр, курс, резерв, лимит, кеш | Открыть проверку ограничений и пройти цепочку |
| Форма показывает неверную сумму | Источник курса, комиссия, selector fee или кратность | Проверить вкладки курсов и комиссий |
| Клиент не может создать заявку | Сумма вне лимитов, реквизит не прошёл проверку, KYC/AML, CAPTCHA | Проверить ошибку на сайте и настройки пары |
| Заявка создаётся без нужного поля | Поле не привязано к валюте или направлению | Проверить поля валюты и направления |
| Мерчант не выбран | Нет правила, валюта не связана с мерчантом, лимит | Проверить правила мерчантов и валюту |
| Автовыплата не выбрана | Нет правила автовыплаты, валюта не связана с выплатой или превышен лимит | Проверить правила выплат и валюту |
| Курс BestChange пропал | Фильтры слишком строгие, черный список/whitelist, нет рынка | Проверить BestChange-логи и фильтры |
