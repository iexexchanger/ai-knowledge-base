# Подробный справочник валют

Валюта в админке — это не просто “BTC” или “USDT”. Валюта управляет тем, как она видна на сайте, какие реквизиты вводит клиент, какая сеть используется, нужна ли верификация, можно ли подключать мерчант, автовыплату, AML и как считать резерв.

## Список валют

| Поле / действие | Что означает | Как использовать | Ошибки при неверной настройке |
|---|---|---|---|
| Поиск по платёжной системе | Фильтр по названию платёжной системы | Введите часть названия, например “Tether” или “Сбер” | Можно не найти валюту, если ищете по коду, а фильтр ждёт платёжную систему |
| Поиск по коду | Фильтр по коду валюты | Введите BTC, USDT, RUB | Если код валюты создан неверно, валюта будет трудно найти |
| XML-код | Код для XML и мониторингов | Используйте при проверке экспортов | Неверный XML-код ломает отображение в мониторингах |
| Фильтр | Группа фильтра в калькуляторе | Проверяйте, почему валюта не видна на сайте | Валюта без нужного фильтра может не попасть в список |
| Сеть | Сеть валюты, например TRC20, ERC20, BEP20 | Используйте для crypto-валют | Неверная сеть приводит к ошибкам платежей |
| Статус | Активная, неактивная, архивная | Активная видна в рабочих сценариях | Архивная или выключенная валюта не должна использоваться в новых заявках |
| Создать валюту | Добавить новую валюту | Сначала создайте код и платёжную систему, затем валюту | Если пара уже существует, создание может быть недоступно |
| Дублировать | Скопировать валюту | Удобно для USDT TRC20 и USDT ERC20 | После дубля обязательно поменяйте сеть, XML и поля |
| Архивировать | Убрать валюту из работы | Используйте вместо удаления, если валюта уже была в заявках | Не архивируйте валюту с активными направлениями без проверки |
| Восстановить | Вернуть валюту из архива | После восстановления проверьте статус и направления | Архивные связи могли устареть |

## Создание валюты

| Поле / настройка | Что означает | Как правильно заполнять | Важно знать |
|---|---|---|---|
| Платёжная система | Логотип и общий тип валюты | Выберите существующую или создайте новую | Платёжная система влияет на иконку и группировку |
| Код валюты | Базовый код, например BTC, USDT, RUB | Выберите из справочника кодов | Один код может использоваться в разных сетях |
| Добавить платёжную систему | Быстрое создание новой системы | Используйте, если нужной системы нет | После создания проверьте логотип |
| Добавить код валюты | Быстрое создание нового кода | Используйте только если кода нет | Не создавайте дубли вроде `USDT2` без необходимости |
| Недавно добавленные | Быстрый выбор новых элементов | Удобно после создания кода или системы | Не означает, что валюта уже настроена |
| Проверка существующей пары | Система предупреждает о дубле | Если пара уже есть, лучше открыть существующую | Дубли могут запутать направления и резервы |

## Основные настройки валюты

| Поле / настройка | Что означает | Как правильно заполнять | Рекомендация |
|---|---|---|---|
| Статус | Рабочее состояние валюты | Активная — используется; неактивная — скрыта; архивная — убрана из работы | Перед выключением проверьте активные направления |
| Платёжная система | К какой системе относится валюта | Выберите правильную систему с иконкой | Влияет на отображение клиенту |
| Код валюты | Денежный код | BTC, USDT, USD, RUB | Используйте стандартные коды |
| Отображать код валюты | Показывать код рядом с названием | Да для криптовалют и похожих валют | Помогает отличать USDT TRC20 от USDT ERC20 |
| Техническое название | Название для админки и клиента | Пишите понятно: “Tether USDT TRC20” | Не используйте внутренние сокращения |
| XML-обозначение | Код для экспорта и мониторингов | Выберите из списка или задайте корректно | Неверный XML ломает внешние выгрузки |
| Формат числа | Количество знаков суммы | Для BTC больше знаков, для RUB меньше | Неверная точность портит суммы |
| Конвертировать по | Через какой код считать конвертацию | Обычно по базовому коду валюты | Важно для резервов, балансов и аналитики |
| Подзаголовок валюты | Текст под названием валюты | Например “сеть TRC20” | Используйте, чтобы клиент не ошибся сетью |
| Сеть | Blockchain или платёжная сеть | TRC20, ERC20, BEP20, BTC, TON | Для crypto обязательно проверить |
| Короткий код сети | Короткое обозначение сети | Например TRC20 | Должно совпадать с тем, что видит клиент |

## Сети и группы валют

| Поле / настройка | Что означает | Как правильно заполнять | Рекомендация |
|---|---|---|---|
| Название сети | Полное название сети | Например “TRON TRC20” | Должно быть понятно клиенту |
| Тип отображения | Как сеть показывается в списке | Стандартно или компактно | Компактно удобно при большом списке |
| Иконка | Изображение сети | Загружайте простую иконку | Не путайте иконку валюты и сети |
| Валюты сети | Какие валюты входят в сеть | Привяжите все USDT/TRX и похожие валюты | Неверная привязка путает клиента |
| Принудительное перемещение | Переместить валюты в эту сеть | Используйте осторожно | Может изменить отображение сразу у многих валют |

## Поля реквизитов валюты

| Поле / настройка | Что означает | Как правильно заполнять | Рекомендация |
|---|---|---|---|
| Доп. поля “Отдаю” | Дополнительные поля для стороны, которую клиент отдаёт | Выберите созданные поля | Не добавляйте лишние поля, они снижают конверсию |
| Доп. поля “Получаю” | Дополнительные поля для стороны, которую клиент получает | Выберите нужные поля | Используйте для банка, ФИО, города, комментария |
| Название поля счёта | Как назвать главное поле реквизита | “Номер карты”, “Адрес кошелька”, “Счёт” | Название должно совпадать с валютой |
| Комментарий поля счёта | Подсказка под полем | Объясните формат и сеть | Здесь можно предупредить про TRC20/ERC20 |
| Скрыть поле “Отдаю” | Не показывать поле реквизита на стороне отдачи | Включайте, если реквизит не нужен | Ошибка может убрать обязательный реквизит |
| Скрыть поле “Получаю” | Не показывать поле реквизита на стороне получения | Включайте только для сценариев без реквизита | Для выплат обычно реквизит нужен |
| Первый символ | Автоподстановка или ограничение первого символа | Используйте для кошельков со стабильным префиксом | Не ставьте, если адреса бывают разные |
| Минимум символов | Минимальная длина реквизита | Укажите реальный минимум | Слишком большой минимум не даст создать заявку |
| Максимум символов | Максимальная длина реквизита | Укажите реальный максимум | Слишком маленький максимум отрежет кошельки |
| Ошибка min/max | Текст при неверной длине | Пишите понятно: “Проверьте длину адреса” | Без понятного текста клиент пишет в поддержку |
| Название поля “Отдаю” | Название для стороны “Отдаю” | Например “Ваш адрес отправителя” | Используйте, если разные подписи по сторонам |
| Название поля “Получаю” | Название для стороны “Получаю” | Например “Ваш адрес USDT TRC20” | Важно для выплат |
| Комментарий “Отдаю” | Подсказка к реквизиту отдачи | Объясните, что именно ввести | Не вставляйте секреты |
| Комментарий “Получаю” | Подсказка к реквизиту получения | Укажите сеть, формат, предупреждение | Это снижает ошибки выплат |
| Маска “Отдаю” | Формат ввода | Для карт можно использовать группировку цифр | Неверная маска не даст ввести реквизит |
| Маска “Получаю” | Формат реквизита получения | Используйте только для предсказуемого формата | Для crypto-адресов маска часто не нужна |
| Placeholder маски | Символ-заполнитель | Обычно `_` или пусто | Не должен путать клиента |
| Валидатор “Отдаю” | Проверка реквизита | Выберите валидатор карты, кошелька, email и т.д. | Неверный валидатор блокирует заявки |
| Валидатор “Получаю” | Проверка реквизита выплаты | Выберите по валюте и сети | Особенно важно для crypto-адресов |
| Ошибка валидатора “Отдаю” | Текст при неверном реквизите | Пишите, что проверить | Например сеть, длину, символы |
| Ошибка валидатора “Получаю” | Текст при неверном реквизите | Пишите без технических терминов | Клиент должен исправить сам |
| Допустимые символы | Какие символы можно вводить | Любые, числа, буквы, латиница, латиница и цифры | Для карт — числа, для crypto — латиница и цифры |
| Удалять пробелы | Автоматически убрать пробелы из реквизита | Да для карт и кошельков | Уменьшает ошибки копирования |
| Разрешённые банки | Банки, которые можно использовать | Укажите список при банковских правилах | Если пусто, правило может не применяться |
| Запрещённые банки | Банки, которые нельзя использовать | Добавляйте только по регламенту | Ошибка заблокирует нормальные карты |
| QR “Отдаю” | Показывать сканирование QR | Да, если клиент может сканировать реквизит | Удобно для кошельков |
| QR “Получаю” | Показывать сканирование QR | Да для адреса получения | Проверьте на мобильном |

## Информация валюты

| Поле / настройка | Что означает | Как правильно заполнять | Рекомендация |
|---|---|---|---|
| Источник инструкции | Откуда брать инструкцию | Автоматически, из направления или из валюты | Для общих правил используйте валюту, для пары — направление |
| Инструкция обмена | Текст инструкции для клиента | Пишите пошагово | Не перегружайте юридическим текстом |
| Источник описания | Откуда брать описание | Авто, направление или валюта | Если у направления особые условия, выбирайте направление |
| Описание обмена | Текст о валюте | Укажите особенности сети и платежа | Помогает избежать ошибок |
| Формализация | Текст оформления или условий | Пишите простым языком | Может показываться в публичной форме |
| Документы “Отдаю” | Текст по документам на стороне отдачи | Заполняйте только если нужны документы | Не просите лишние данные |
| Документы “Получаю” | Текст по документам на стороне получения | Заполняйте по требованиям выплаты | Должно совпадать с KYC |
| Предупреждение “Отдаю” | Короткое предупреждение в форме | Например “Отправляйте только TRC20” | Важно для crypto |
| Предупреждение “Получаю” | Короткое предупреждение выплаты | Например “Проверьте сеть адреса” | Ошибки сети необратимы |
| Название кнопки | Текст кнопки создания заявки | Обычно “Обменять” или “Создать заявку” | Не делайте длинным |
| Текст под кнопкой | Дополнительное пояснение | Укажите важные условия | Не скрывайте комиссии |
| Цвет текста | Цветовое оформление валюты | Используйте умеренно | Цвет не должен ухудшать читаемость |

## Мерчанты валюты

| Поле / настройка | Что означает | Как правильно заполнять | Рекомендация |
|---|---|---|---|
| Список мерчантов | Какие провайдеры могут принимать оплату по валюте | Выберите рабочие мерчанты | Не выбирайте непроверенные |
| Сеть мерчанта по умолчанию | Какая сеть отправляется мерчанту | Например TRC20 | Должна совпадать с валютой |
| Сеть по каждому мерчанту | Индивидуальная сеть для провайдера | Заполняйте, если провайдер ждёт свой код | Ошибка сети приводит к невыдаче реквизитов |
| Валидатор мерчанта | Проверка реквизитов от мерчанта | Выберите, если нужно проверять выданные реквизиты | Полезно против ошибочных карт/адресов |
| Стратегия ошибки валидатора | Что делать, если реквизит не прошёл проверку | Игнорировать или показать ошибку | Для финансовых реквизитов лучше останавливать |
| Суточный объём | Максимальная сумма через мерчант за день | `0` обычно значит без лимита | Лимиты защищают от перегрузки провайдера |
| Месячный объём | Максимальная сумма за месяц | Укажите по договору с провайдером | При превышении мерчант станет недоступен |
| Минимум на заявку | Мин. сумма одной заявки | Ставьте по лимитам провайдера | Ниже лимита реквизиты не выдадутся |
| Максимум на заявку | Макс. сумма одной заявки | Ставьте по лимитам провайдера | Выше лимита клиент получит ошибку |
| Дневной лимит заявок | Количество заявок в день | `0` без лимита | Полезно при лимитах провайдера |
| Месячный лимит заявок | Количество заявок в месяц | Укажите по регламенту | Не путать с лимитом суммы |

## AML валюты

| Поле / настройка | Что означает | Как правильно заполнять | Рекомендация |
|---|---|---|---|
| Источник AML | Провайдер проверки | Выберите активный AML-сервис | Без сервиса проверка не работает |
| Проверка счёта | Проверять кошелёк до заявки или при проверке оплаты | Нет, перед созданием, при проверке оплаты | Для crypto-входа часто проверяют до создания или при оплате |
| Сумма обменов “От” для кошелька | Минимальная сумма, с которой включать проверку | `0` может означать без ограничения | Не проверяйте мелкие суммы без необходимости |
| Если превышен риск кошелька | Что делать при высоком риске | Ничего не делать или выдать ошибку | Для строгого AML выбирайте ошибку |
| Проверка транзакции | Проверять конкретную транзакцию | Нет, при проверке оплаты, во время автовыплаты | Транзакционная проверка точнее |
| Сумма “От” для транзакции | Порог AML-проверки транзакции | Укажите сумму риска | Согласуйте с AML-политикой |
| Если превышен риск транзакции | Реакция на высокий риск | Ничего не делать или выдать ошибку | При ошибке клиент может не пройти дальше |
| AML текст “Отдаю” | Текст для клиента на стороне оплаты | Объясните проверку риска | Не пишите внутренние пороги |
| AML текст “Получаю” | Текст для клиента на стороне выплаты | Объясните возможную проверку | Полезно для crypto-выплат |

## Лимиты резерва валюты

| Поле / настройка | Что означает | Как правильно заполнять | Рекомендация |
|---|---|---|---|
| Максимальный лимит в резерве | Верхний предел резерва | Укажите реальный лимит | Защищает от завышенного отображения |
| Максимальный отображаемый резерв | Сколько показывать клиенту | Можно показать меньше фактического | Полезно для приватности |
| Процент прибыли резерва | Процент для расчёта резерва/прибыли | Заполняйте только если используется схема | Ошибка влияет на аналитику |
| Процент переноса резерва | Как переносить часть резерва | Используйте при связанных резервах | Требует ручной проверки |
| Сумма переноса резерва | Фиксированная сумма переноса | Заполняйте по регламенту | Не используйте без понимания |

## Частые проблемы по валютам

| Ситуация | Причина | Что сделать |
|---|---|---|
| Валюта не видна на сайте | Статус выключен, нет направления, не тот фильтр, кеш | Проверить статус, направление, фильтр и очистить кеш |
| Клиент не может ввести реквизит | Маска, min/max, валидатор или допустимые символы слишком строгие | Проверить поля валюты |
| Клиент отправил не в ту сеть | Непонятное название, нет предупреждения, неверная сеть | Исправить сеть, подзаголовок и notice |
| Мерчант не выдаёт реквизиты | Не выбран мерчант, лимит, сеть или ошибка валидатора | Проверить мерчанты валюты и журналы платежных шлюзов |
| Автовыплата не уходит | Не выбрана выплата, allow_autopay, лимит или сеть | Проверить выплаты валюты |
| AML блокирует все заявки | Слишком низкий порог или включена ошибка при риске | Проверить AML-настройки валюты |
