# Партнерская программа: запуск, настройки и диагностика

Партнерская программа позволяет начислять бонусы партнерам за клиентов и заявки. Чтобы она работала предсказуемо, нужно настроить три слоя: публичные условия, реферальные программы и глобальные настройки начисления.

## Что входит в партнерский раздел

| Раздел | Для чего нужен |
|---|---|
| Условия | Тексты для партнеров, скидок и мониторингов |
| Реферальные программы | Проценты, названия, описания, lifetime |
| Настройки партнерки | Включение системы, логирование, cookie, минимальная выплата, программа по умолчанию |
| Скидки | Отдельная система скидок/cashback |
| Баннеры | Публичные партнерские баннеры |
| Реферальные логи | История регистрации, привязки, начисления, ошибок и HOLD |
| Аналитика партнеров | Партнерские заявки, ROI, бонусы, качество трафика |

## Быстрый порядок запуска

1. Заполните текст "Реферальная программа" в условиях.
2. Создайте минимум одну реферальную программу.
3. Укажите понятное название и описание.
4. Укажите процент партнера.
5. Откройте настройки партнерки.
6. Включите реферальную программу.
7. Выберите программу по умолчанию.
8. Включите логирование.
9. Укажите тип начисления: от суммы обмена или от прибыли направления.
10. Настройте срок cookie.
11. Укажите минимальную сумму выплаты.
12. Сделайте тестовую регистрацию по партнерской ссылке.
13. Создайте тестовую заявку и проверьте реферальный лог.

## Условия

| Поле | Что означает | Как заполнять |
|---|---|---|
| Реферальная программа | Публичное объяснение партнерки | Как стать партнером, как считается бонус, когда он доступен |
| Скидки | Текст про скидки/cashback | Укажите реальные условия, не обещайте лишнее |
| Мониторинг | Текст для мониторингов и партнерских площадок | Опишите правила сотрудничества |

Тексты условий должны совпадать с реальными настройками. Если написано "партнер получает 2% от обмена", а в настройках выбран расчет от прибыли, партнеры будут задавать вопросы.

## Реферальные программы

| Поле | Как работает | Пример |
|---|---|---|
| Название | Название программы по языкам | `Базовая 2%`, `VIP 5%` |
| Описание | Условия программы | "Начисление после успешного обмена" |
| Процент | Процент вознаграждения | `2`, `0.5`, `5` |
| Lifetime minutes | Срок действия программы/привязки в минутах | 5 месяцев, 1 год, 3 года |

### Как выбирать процент

| Модель | Какой процент ставить | Когда подходит |
|---|---|---|
| Новая партнерка | Небольшой процент | Чтобы проверить качество трафика |
| VIP-партнеры | Более высокий процент | Для проверенных партнеров |
| Трафик с высоким риском | Низкий процент или отдельная программа | Если много отказов, AML, дублей |
| Расчет от прибыли | Процент может быть выше | Потому что база расчета уже меньше оборота |

## Глобальные настройки партнерки

| Настройка | Как работает | Рекомендация |
|---|---|---|
| Минимальная выплата | Порог, с которого партнер может запросить вывод | Ставьте понятную сумму по регламенту |
| Включить реферальную программу | Разрешает новые начисления | Должно быть включено для работы партнерки |
| Программа по умолчанию | Программа для новых партнеров | Обязательно выбрать рабочую программу |
| Логировать реферальные события | Записывает события в журнал | Включайте всегда, иначе споры трудно проверить |
| Тип вознаграждения | От суммы обмена или от прибыли направления | Выберите бизнес-модель до запуска |
| Срок хранения/cookie | Без ограничения или ограничить по дням | Для рекламы часто ставят 90/365/1825 дней |
| Количество дней cookie | Сколько дней хранить привязку | Используется только если срок ограничен |

### Тип начисления

| Тип | Что означает | Плюсы | Риск |
|---|---|---|---|
| От суммы обмена | Процент считается от оборота заявки | Просто объяснить партнеру | Может быть дорого при маленькой марже |
| От прибыли направления | Процент считается от прибыли обменника | Точнее для экономики | Партнеру сложнее понять расчет |

Если выбран расчет от прибыли, проверьте настройки прибыли направления. Если по заявке прибыль равна 0, партнерский бонус тоже может стать 0.

## Срок cookie и привязка клиента

Cookie определяет, как долго клиент помнит партнерскую ссылку.

| Настройка | Как работает |
|---|---|
| Без ограничения | Привязка не ограничена сроком в днях |
| Ограничить по времени | Привязка действует указанное количество дней |
| 90 дней | Хорошо для рекламных кампаний |
| 365 дней | Частый вариант для долгой партнерки |
| 1825 дней | Почти долгосрочная привязка |

Если клиент уже привязан к одному партнеру, новая ссылка может не перезаписать старую привязку. Это зависит от правил системы и должно проверяться по журналу.

## Реферальные логи

Реферальные логи — главный инструмент диагностики партнерки.

| Фильтр | Когда использовать |
|---|---|
| Search | Если есть текст ошибки или событие |
| Event | Когда нужно конкретное событие |
| Level | Чтобы найти warning/error |
| Partner ID | Спор конкретного партнера |
| Client ID | Спор по конкретному клиенту |
| Task ID | Спор по конкретной заявке |
| Referral link ID | Проверка рекламной ссылки |
| Referral program ID | Проверка программы |
| Trace ID | Глубокая цепочка событий |
| Date from/to | Ограничить период |

### События

| Событие | Что означает | Что делать |
|---|---|---|
| register_start | Клиент начал регистрацию | Проверить, завершил ли регистрацию |
| register_finish | Регистрация завершена | Проверить `client_attached` |
| client_attached | Клиент закреплен за партнером | Привязка сработала |
| program_assigned | Назначена программа | Проверить процент программы |
| balance_created | Создан баланс партнера | Нормальное служебное событие |
| credit_success | Бонус начислен | Проверить сумму и заявку |
| credit_ineligible | Начисление недоступно | Читать причину в контексте |
| credit_duplicate | Повторное начисление пропущено | Защита от дубля |
| credit_zero | Сумма бонуса 0 | Проверить процент, прибыль, тип начисления |
| hold_settled | HOLD снят, бонус доступен | Можно сообщить партнеру |
| credit_reversed | Бонус отменен | Проверить причину сторно |
| preview | Предварительный расчет | Это не реальное начисление |

## Почему бонус не начислился

| Причина | Что проверить |
|---|---|
| Партнерка выключена | Настройки партнерки |
| Нет программы по умолчанию | Выбрана ли программа |
| У партнера нет программы | Профиль партнера/программа |
| Клиент не привязан | Событие `client_attached` |
| Cookie истек | Срок cookie |
| Заявка не завершена | Статус заявки |
| Бонус в HOLD | Событие `hold_settled` еще не было |
| Повторное начисление | `credit_duplicate` |
| Бонус 0 | Процент, прибыль, тип начисления |
| Логи выключены | Включить логирование и повторить тест на новой заявке |

## Как проверить конкретную заявку

1. Возьмите ID заявки.
2. Откройте карточку заявки и проверьте статус.
3. Проверьте, есть ли партнер или реферальная ссылка.
4. Откройте "Журналы -> Реферальные события".
5. Поставьте фильтр `task_id`.
6. Посмотрите события `credit_success`, `credit_zero`, `credit_ineligible`, `credit_duplicate`.
7. Если бонус в HOLD, проверьте условия разблокировки.
8. Если нет событий, проверьте включение логов и привязку клиента.

## Скидки

Скидки — отдельная функция. Их не нужно путать с партнерскими бонусами.

| Настройка | Как работает |
|---|---|
| Скидки включены | Система скидок может применяться |
| Скидки выключены | Скидки не должны применяться |

Если клиент спрашивает про cashback или скидку, проверьте не только партнерку, но и раздел скидок, условия клиента, направление и статус заявки.

## Баннеры партнеров

| Поле | Как работает | Рекомендация |
|---|---|---|
| Название | Название партнера | Заполнить основной язык |
| Ссылка | Куда ведет баннер | Только корректный http/https |
| Логотип | Изображение партнера | Легкий, четкий файл |
| Статус | Показывать или скрыть | Выключайте временно, не удаляйте сразу |
| Сортировка | Порядок на сайте | Важные партнеры выше |

## Аналитика партнеров

| Раздел | Что смотреть |
|---|---|
| Overview | Активные партнеры, заявки, новые рефералы, бонусы |
| Links | Эффективность ссылок |
| Referrals | Список привлеченных клиентов |
| Finance | Начисления, бонусы, баланс |
| Partner ROI | Доходность партнеров |
| Partner quality | Завершенные и проблемные заявки |
| Partner programs | Сравнение программ |

Не оценивайте партнера только по обороту. Смотрите завершенные заявки, проблемные заявки, прибыль и AML/KYC-риск.

## Ответы на частые вопросы

| Вопрос | Ответ |
|---|---|
| Где включить партнерку? | "Партнеры -> Реферальные программы -> настройки, включите реферальную систему и выберите программу по умолчанию." |
| Почему партнер не получил процент? | "Проверьте статус заявки, привязку клиента, программу, тип начисления и реферальные логи по task_id." |
| Почему клиент не закрепился? | "Проверьте cookie, старую привязку, ссылку партнера и событие client_attached." |
| Что лучше: от суммы или от прибыли? | "От суммы проще объяснить, от прибыли безопаснее для экономики обменника." |
| Можно ли сделать VIP-процент? | "Да, создайте отдельную программу и назначьте ее нужному партнеру." |
| Почему бонус 0? | "Чаще всего прибыль по заявке 0, процент 0 или условия начисления не выполнены." |

## Важные предупреждения

Не меняйте тип начисления на рабочей партнерке без предупреждения партнеров. Это может изменить будущие выплаты.

Не выключайте реферальные логи. Без них почти невозможно доказать, почему бонус начислился или не начислился.

Не обещайте партнерам условия, которые не отражены в настройках и публичных текстах.
