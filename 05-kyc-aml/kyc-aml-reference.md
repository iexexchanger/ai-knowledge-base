# KYC, верификация карт и AML

Этот справочник нужен для вопросов о проверке карт, проверке личности, AML-проверке кошельков и транзакций, ручных формах KYC, сервисах и логах.

## Верификация карты

| Поле / настройка | Что означает | Как правильно заполнять | Рекомендация |
|---|---|---|---|
| Список заявок на проверку | Клиенты, которые отправили карту на проверку | Фильтруйте по статусу и дате | Начинайте с ожидающих |
| Статус | Ожидает, одобрена, отклонена | Меняйте только после просмотра материалов | Ошибка влияет на возможность обмена |
| Категория | Тип инструкции или проверки | Выберите подходящую категорию | Помогает клиенту понять требования |
| Инструкция | Что должен сделать клиент | Пишите пошагово | Укажите качество фото и что закрывать нельзя |
| Текст уведомления | Короткое пояснение | Просто и без давления | Клиент должен понимать следующий шаг |
| Количество неудачных попыток | Сколько раз клиент может ошибиться | Укажите разумный лимит | Слишком мало попыток создаёт обращения |
| Автоверификация карты | Система пытается проверить автоматически | Включайте только при настроенном процессе | Нужна ручная проверка ошибок |
| Описание проверки | Текст перед отправкой | Объясните зачем проверка | Не просите лишнее |
| Текст ошибки проверки | Что показать при отказе | Пишите понятно: “Фото не читается” | Не раскрывайте внутренние правила риска |

## Категории верификации карт

| Поле / настройка | Что означает | Как правильно заполнять | Рекомендация |
|---|---|---|---|
| Название категории | Группа инструкций | Например “Банковская карта”, “Виртуальная карта” | Простое название |
| Статус | Категория активна | Включайте только готовые категории | Выключенная категория не используется |
| Сортировка | Порядок категорий | Важные выше | Удобно для поддержки |

## Инструкции верификации карт

| Поле / настройка | Что означает | Как правильно заполнять | Рекомендация |
|---|---|---|---|
| Статус инструкции | Показывать или нет | Включайте только проверенный текст | Старые инструкции скрывайте |
| Категория | Где инструкция используется | Выберите правильную категорию | Иначе клиент увидит не то |
| Название | Заголовок инструкции | Коротко и понятно | Например “Фото карты на фоне заявки” |
| Текст инструкции | Подробные шаги | Пишите по пунктам | Укажите, какие цифры можно закрыть |
| Notice | Короткое предупреждение | Используйте для важных условий | Не делайте слишком длинным |

## KYC заявок личности

| Поле / настройка | Что означает | Как правильно заполнять | Рекомендация |
|---|---|---|---|
| Список KYC | Все проверки личности | Фильтруйте ожидающие и спорные | Не раскрывайте данные клиентам |
| Статус KYC | Ожидает, одобрено, отклонено | Меняйте после проверки документов или ответа сервиса | Нужна причина отказа |
| Пользователь | Чей аккаунт проверяется | Откройте профиль при споре | Сравните с заявками |
| Сервис KYC | Через кого проводится проверка | Выберите активный сервис | Без сервиса проверка не стартует |
| Applicant | Заявитель у провайдера | Используйте для поиска в логах | Может быть внутренним ID провайдера |
| Логи KYC | Ответы сервиса | Смотрите при зависании | Могут содержать чувствительные данные |
| Ручная форма | KYC без внешнего провайдера | Используйте при ручной проверке | Нужно хранить данные безопасно |

## Настройки KYC

| Поле / настройка | Что означает | Как правильно заполнять | Рекомендация |
|---|---|---|---|
| Включить KYC для обмена | Требовать проверку личности в сценариях обмена | Включайте по AML/KYC политике | Может добавить шаг проверки в форму заявки |
| KYC в заявке | Показывать проверку прямо в процессе заявки | Включайте при поддержанном сервисе | Проверьте публичную форму заявки |
| Цель проверки | Цель проверки | Укажите понятную цель | Может отображаться у провайдера |
| Тип сервиса | Какой KYC-сервис используется | Выберите из доступных | Должен совпадать с подключенным сервисом |
| KYC-сервис | Какой сервис использовать | Выберите активный сервис | Сервис должен быть заполнен и включён |

## KYC-сервисы

| Поле / настройка | Что означает | Как правильно заполнять | Рекомендация |
|---|---|---|---|
| Название | Имя сервиса в админке | Например Sumsub или ручная KYC-проверка | Понятно для команды |
| Тип сервиса | Тип сервиса | Выберите из доступных | Нельзя менять без понимания |
| Статус | Активен ли сервис | Включайте после теста | Выключенный сервис не запускает KYC |
| Поля подключения | Поля подключения | Заполняйте по данным провайдера | Секреты не добавлять в документацию |
| Настройки сервиса | Поведение сервиса | Настраивайте по требованиям KYC | Проверяйте после изменения |
| Ручная форма | Ручная форма проверки | Настраивайте поля, если KYC делается вручную | Собирайте минимум данных |

## Конструктор ручной KYC-формы

| Поле / настройка | Что означает | Как правильно заполнять | Рекомендация |
|---|---|---|---|
| Название формы | Заголовок формы | Пишите просто | Например “Проверка личности” |
| Описание формы | Что должен сделать клиент | Объясните, какие документы нужны | Укажите срок проверки |
| Ключ поля | Внутреннее имя поля | Латиница без пробелов | Не меняйте после запуска без необходимости |
| Тип поля | Текст, файл, дата, выбор | Выберите по данным | Для документов часто нужен файл |
| Название поля | Что видит клиент | “Фото паспорта”, “Дата рождения” | Не используйте внутренние термины |
| Placeholder | Пример заполнения | Укажите формат | Например `ДД.ММ.ГГГГ` |
| Help text | Подсказка | Объясните требования к файлу | Уменьшает отказы |
| Обязательное | Нужно ли поле | Включайте только необходимое | Лишние поля ухудшают прохождение |
| Сортировка | Порядок полей | Документы и контакты в логичном порядке | Не смешивайте разные темы |

## KYC в валюте и направлении

| Где настраивается | Что означает | Как выбрать |
|---|---|---|
| Валюта — Верификация личности | Правило применяется ко всем направлениям с этой валютой | Используйте, если сама валюта рискованная |
| Направление — Верификация личности | Правило применяется только к конкретной паре | Используйте, если риск зависит от пары |
| Режим выключено | KYC не требуется | Для низкорисковых направлений |
| Режим всегда | KYC нужен для любой суммы | Для строгих направлений |
| Режим от суммы | KYC нужен после порога | Хороший баланс удобства и контроля |

## AML-сервисы

| Поле / настройка | Что означает | Как правильно заполнять | Рекомендация |
|---|---|---|---|
| Название | Имя AML-сервиса | Например AMLBot, BitOK | Понятно для команды |
| Тип провайдера | Тип провайдера | Выберите из доступных | Должен совпадать с выбранным сервисом |
| Статус | Активен ли сервис | Включайте после проверки подключения | Выключенный сервис не проверяет |
| Поля подключения | Данные подключения | Заполняйте в админке | Не документируйте ключи |
| Настройки проверки | Настройки проверки | По политике риска | Проверьте тестовую проверку |

## AML валюты

| Поле / настройка | Что означает | Как правильно заполнять | Рекомендация |
|---|---|---|---|
| Источник AML | Какой сервис проверяет | Выберите активный AML-сервис | Без сервиса проверка не работает |
| Проверка счёта | Проверять адрес кошелька | Нет, до заявки, при проверке оплаты | До заявки быстрее показывает клиенту проблему |
| Сумма для проверки счёта | От какой суммы проверять | Укажите порог | Слишком низко увеличит расходы и задержки |
| Реакция на риск счёта | Что делать при превышении риска | Ничего или ошибка | Ошибка блокирует сценарий |
| Проверка транзакции | Проверять конкретный перевод | При оплате или автовыплате | Точнее, чем проверка адреса |
| Сумма для проверки транзакции | От какой суммы проверять | Укажите по политике | Учитывайте стоимость проверок |
| Реакция на риск транзакции | Что делать при высоком риске | Ничего или ошибка | Для строгого AML — ошибка и ручная проверка |
| AML текст “Отдаю” | Сообщение для клиента | Объясните проверку | Не пишите внутренние пороги риска |
| AML текст “Получаю” | Сообщение для выплаты | Объясните возможную проверку | Особенно для crypto |

## AML-логи

| Поле | Что означает | Как использовать |
|---|---|---|
| ID заявки | К какой заявке относится проверка | Ищите по заявке |
| Тип проверки | Кошелёк или транзакция | Понимание стадии |
| Провайдер | Какой AML-сервис отвечал | Если ошибка, проверяйте сервис |
| Уровень риска | Уровень риска | Сравнивайте с политикой |
| Статус | Успешно, ошибка, требуется действие | Решение зависит от статуса |
| Ответ провайдера | Подробности проверки | Только для внутренней диагностики |
| Дата | Когда проверяли | Сравнивайте с оплатой |

## Как выбрать между KYC и AML

| Ситуация | Что использовать | Почему |
|---|---|---|
| Нужно проверить личность клиента | KYC | Проверяет человека и документы |
| Нужно проверить crypto-адрес | AML wallet check | Проверяет риск адреса |
| Нужно проверить конкретный перевод | AML transaction check | Точнее по фактической транзакции |
| Нужна проверка карты | Верификация карты | Проверяет владение картой |
| Клиент часто создаёт подозрительные заявки | Антиспам и KYC/AML по регламенту | Не заменяйте одно другим |

## Типовые ошибки

| Ситуация | Причина | Что сделать |
|---|---|---|
| KYC не запускается | Сервис не выбран, выключен, неверные поля подключения | Проверить KYC-сервис и настройки |
| KYC завис | Провайдер не вернул финальный статус или очередь стоит | Проверить KYC applicants/журналы и очереди |
| Клиент не видит форму KYC | Правило не включено для валюты/направления или сайт показывает старые данные | Проверить валюту, направление, кеш |
| AML блокирует все заявки | Слишком строгие правила или ошибка провайдера трактуется как блокировка | Проверить AML-сервис и реакцию на риск |
| AML не проверяет | Не выбран сервис, сумма ниже порога, проверка выключена | Проверить валюту и сумму заявки |
| Карта не проходит проверку | Фото плохое, категория не та, попытки исчерпаны | Дать клиенту понятную инструкцию |
| Оператор не видит KYC/AML | Нет права роли | Проверить доступы |

## Безопасные рекомендации

1. Собирайте только те документы и данные, которые реально нужны.
2. Не отправляйте клиенту сырые AML/KYC-ответы провайдера.
3. Причины отказа пишите понятно, без раскрытия внутренних risk score.
4. При споре сохраняйте внутренний комментарий и смотрите логи.
5. Не добавляйте в документацию ключи AML/KYC-провайдеров, реальные документы, фото, адреса, телефоны, email или приватные URL.
