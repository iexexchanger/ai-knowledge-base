# Маршрутизатор задач для AI-чата

## Назначение

Эта страница помогает AI-агенту быстро понять, в какой раздел админки направить пользователя и какие проверки предложить.

Страница написана для чата. Ее задача — не описывать код, а превращать вопрос пользователя в понятные действия.

## Как пользоваться в чате

1. Определить тему вопроса: курс, направление, заявка, пользователь, платеж, KYC, AML, уведомления, безопасность или контент.
2. Назвать раздел админки, где это настраивается.
3. Дать 3-7 проверок по порядку.
4. Предупредить, если настройка влияет на деньги, безопасность или активные заявки.
5. Если данных мало, задать один уточняющий вопрос.

| “Что писать в API_URL” | Nginx для Angular SSR | Backend-origin для SSR, `/backend-api` только для browser path | Средний |

## Маршруты по настройкам

| Задача | Раздел | Минимальный порядок проверки | Что нельзя делать |
| --- | --- | --- | --- |
| Добавить новую валюту | Валюты | Создать валюту, настроить сеть, поля, статус, резерв | Включать без реквизитов и теста |
| Добавить новое направление | Направления | Валюты, курс, лимиты, резерв, реквизиты, проверки, тест | Создавать направление без проверки формы сайта |
| Изменить курс | Курсы или направление | Тип курса, источник, формула, наценка, preview | Менять рабочую формулу без теста |
| Изменить резерв | Резервы | Источник резерва, сумма, дата обновления, связанная пара | Завышать резерв без реальных средств |
| Включить KYC | KYC и направление | Сервис, условия, тексты, ручная проверка | Включать без инструкции клиенту |
| Включить AML | AML и валюта | Сервис, пороги, криптовалюта, действие при риске | Продолжать рискованную заявку без регламента |
| Подключить мерчант | Платежные системы | Провайдер, валюта, лимиты, тест, журнал | Публиковать ключи и приватные данные |
| Включить автовыплату | Автовыплаты | Провайдер, резерв, лимит, тестовая выплата, журнал | Включать на большие суммы без теста |
| Настроить письмо | Email | Событие, шаблон, язык, тест | Удалять рабочий шаблон без копии |
| Настроить Telegram | Telegram Bot | Бот, канал, событие, шаблон, тест | Хранить токен в документации |
| Создать роль оператора | Роли | Только заявки, чат, нужные журналы | Давать доступ к выплатам и настройкам без причины |
| Изменить дизайн | Дизайн | Тема, логотип, блоки, мобильная проверка | Удалять старые изображения до проверки |
| Изменить текст правил | Контент | Страница, язык, статус, ссылка, чекбокс | Менять юридический текст без владельца |
| Начать настройку с нуля | С чего начать настройку админки | Сервер, доступ, основные настройки, уведомления, первая валюта, первое направление, тестовая заявка | Запускать клиентов до теста |
| Подготовить сервер | Техническая настройка | FastPanel/Nginx, PHP 8.4, база, Redis, cron `/opt/php84/bin/php .../artisan schedule:run`, Horizon, права, backup | Считать сайт готовым без фоновых процессов |
| Настроить frontend-домен | Nginx для Angular SSR | SSR upstream, `/backend-api`, backend Host/SNI, static aliases, `/ng-custom/` | Направлять browser API напрямую на backend-домен |

## Диагностика по симптомам

| Симптом | Вероятная область | Проверка | Когда нужен специалист |
| --- | --- | --- | --- |
| На сайте нет всех направлений | Валюты, направления, кеш | Статусы, резервы, курсы, лимиты | Если после проверки все включено, но сайт пустой |
| Сайт принимает заявки, но операторы их не видят | Заявки, роли, фильтры | Фильтр списка, права, статус заявки | Если новые заявки не появляются ни у кого |
| Курсы перестали обновляться во всех парах | Курсы, фоновые процессы | Источники, последний запуск, журналы | Если автообновление не запускалось |
| Один источник курса пустой | Источник курса | Пара, доступность источника, лимиты, прокси | Если источник не отвечает долго |
| Все письма перестали отправляться | Email, очередь | Почтовый сервис, шаблоны, очередь | Если очередь не обрабатывается |
| Все Telegram-сообщения перестали приходить | Telegram, очередь | Бот, канал, права, очередь | Если бот не отвечает |
| Не проходят автовыплаты по всем провайдерам | Выплаты, очередь, резервы | Очередь, лимиты, доступность провайдеров | Если есть риск дубля выплат |
| Только один клиент не может создать заявку | Пользователь, направление | Блокировка, KYC, AML, лимиты, поля | Если нет видимой причины |
| Только один оператор не видит данные | Роли, фильтры, сессия | Права, фильтры, активная сессия | Если права корректные, но доступ не появился |
| Админка стала медленной | Система, журналы, фоновые процессы | Нагрузка, очереди, последние изменения | Если проблема повторяется у всех |
| Сайт открывается, но письма/курсы/выплаты стоят | Серверные фоновые процессы | Cron, Redis, Horizon, очереди, логи задач | Если остановились все автоматические процессы |

## Что уточнять у пользователя

| Тема | Один полезный вопрос |
| --- | --- |
| Валюта | “Какая именно валюта или сеть не отображается?” |
| Направление | “Какая пара, например BTC - USDT, и какая сумма?” |
| Курс | “Какой тип курса выбран: ручной, источник, формула, файл или BestChange?” |
| Формула | “Какой итог показывает preview и какие теги в формуле пустые?” |
| Заявка | “Какой номер заявки и текущий статус?” |
| Платеж | “Оплата входящая или выплата клиенту?” |
| KYC | “Какой статус проверки и какая причина отказа показана?” |
| AML | “Проверяется адрес, транзакция или заявка целиком?” |
| Уведомления | “Не пришло письмо, Telegram или уведомление в админке?” |
| Доступ | “Это проблема у одного сотрудника или у всех?” |
| Старт настройки | “Это первый запуск после установки или уже рабочий обменник?” |
| Сервер | “Используется FastPanel или ручная настройка Nginx/systemd?” |
| Frontend Nginx | “У вас один домен Laravel или отдельные frontend и backend/admin домены?” |
| Фоновые процессы | “Что именно не запускается: курсы, письма, Telegram, выплаты, архив или аналитика?” |

## Уровни риска ответа

| Уровень | Что означает | Как отвечать |
| --- | --- | --- |
| Низкий | Контент, текст, сортировка, внешний вид | Дать шаги и напомнить проверить сайт |
| Средний | Роли, уведомления, лимиты, промокоды, KYC | Дать шаги, предупредить о последствиях |
| Высокий | Курсы, резервы, платежи, автовыплаты, AML, безопасность | Дать пошаговую проверку, рекомендовать тест и ручное подтверждение |
