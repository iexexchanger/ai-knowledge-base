# Карта покрытия базы знаний для AI-чата

## Для чего нужен раздел

Карта покрытия показывает, какие части админ-панели описаны для AI-чата и какие вопросы должен понимать GitBook Assistant. Это контрольный список для владельца проекта и поддержки: если в админке появляется новая функция, она должна попасть в эту карту и в связанные разделы.

## Основные области

| Область | Что должен понимать AI-агент | Типовые вопросы |
| --- | --- | --- |
| Вход и безопасность | Вход в админку, 2FA, IP-ограничения, сессии, роли | "Почему нет доступа?", "Как добавить менеджера?", "Что делать если потерян 2FA?" |
| Рабочий стол | Счетчики, уведомления, быстрые переходы, поиск | "Что означают счетчики?", "Где посмотреть новые заявки?" |
| Полная карта админки | Реальные разделы меню, назначение каждого блока и связи между ними | "Где это находится?", "Что проверять первым?", "Почему раздел не виден?" |
| Валюты | Коды, сети, поля, фильтры, логотипы, платежные системы, KYC, AML, выплаты | "Как добавить USDT TRC20?", "Почему валюта не видна?" |
| Направления | Пары обмена, курсы, лимиты, комиссии, резервы, сортировка, массовые действия | "Почему BTC - USDT не работает?", "Где поменять лимит?" |
| Комиссии и ограничения направления | Прибыль, доп. комиссии, комиссии ПС, диапазоны суммы, FIX/FLOATING, страны, IP, KYC, лимит новичка и лимиты повторных заявок | "Где поставить комиссию ПС?", "Как включить плавающий курс?", "Почему клиенту закрыто направление?" |
| Реквизиты | Ручные реквизиты, поля, информационные поля, выдача по запросу, архив | "Почему клиент не получил реквизиты?", "Как добавить реквизит?" |
| Резервы | Ручной резерв, файл резервов, резерв направления, скрытие при недостатке | "Почему недостаточно резерва?", "Почему резерв отличается?" |
| Курсы | Источники, формулы, файл, BestChange, конкуренты, автообновление | "Почему курс старый?", "Как настроить формулу?" |
| Заявки | Список, Live-очередь, карточка, статусы, причины, этапы, пересчет, чат, комментарии | "Клиент оплатил, что делать?", "Почему заявка зависла?" |
| Платежные системы | Мерчанты, прием оплаты, журналы, ошибки провайдеров | "Почему мерчант не выдал реквизиты?", "Где смотреть ошибку?" |
| Автовыплаты | Провайдеры выплат, правила, лимиты, ошибки, повторные выплаты | "Почему автовыплата не ушла?", "Можно ли повторить выплату?" |
| KYC | Верификация личности, ручная форма, внешний сервис, статусы, причины отказа | "Почему клиенту нужен KYC?", "Почему KYC не обновляется?" |
| Верификация карт | Категории, инструкции, ручное подтверждение, отказ | "Как подтвердить карту?", "Почему карта отклонена?" |
| AML | Проверка адресов и транзакций, риск, блокировка заявки, журналы | "Почему AML заблокировал заявку?", "Что значит высокий риск?" |
| Пользователи | Профиль, блокировка, балансы, группы, вход через сервисы, дополнительные поля | "Как найти пользователя?", "Почему пользователь заблокирован?" |
| Роли и права | Группы, права разделов, безопасная выдача доступа | "Как дать доступ только к заявкам?", "Почему раздел не виден?" |
| Партнеры | Реферальные программы, бонусы, баннеры, условия, логи | "Почему партнеру не начислился бонус?", "Как задать процент?" |
| Скидки и бонусы | Скидки пользователей, уровни, бонусный центр, промо-условия | "Почему скидка не применилась?", "Как настроить уровень?" |
| Контент сайта | Страницы, меню, новости, FAQ, контакты, преимущества, статистика | "Почему страница не видна?", "Где изменить контакты?" |
| Дизайн | Логотипы, баннеры, визуальные блоки, дизайн-конструктор | "Где заменить логотип?", "Почему баннер не показывается?" |
| Маркетинг | Промо-коды, баннеры, конкурсы | "Почему промо-код не работает?", "Где посмотреть участников конкурса?" |
| Уведомления | Email, Telegram, центр уведомлений, события, шаблоны | "Почему письмо не пришло?", "Почему Telegram молчит?" |
| Аналитика | Прибыль, заявки, направления, пользователи, партнеры, трафик, системные показатели | "Где смотреть прибыль?", "Какие направления популярны?" |
| Журналы | Входы, статусы заявок, AML, KYC, мерчанты, автовыплаты, реквизиты | "Где найти ошибку по заявке?", "Кто изменил статус?" |
| CRON и процессы | Автообновление курсов, очереди, уведомления, фоновые задачи | "Почему курсы не обновляются?", "Почему очередь стоит?" |
| После покупки и запуск | Что входит после покупки, какие доступы нужны, кто что делает, как подготовить домен и сервер | "Какие доступы нужны?", "Можно без домена?", "Можете установить FastPanel?" |
| Запуск обменника по шагам | Полная цепочка запуска от сервера до тестовой заявки | "С чего начать настройку?", "Что проверить перед запуском рекламы?" |
| С чего начать настройку | Порядок действий для новичка: что делать первым, вторым и что проверять после каждого шага | "Я новичок, с чего начать?", "Что настраивать сначала?", "Когда делать тестовую заявку?" |
| Карта связей настроек | Как настройки влияют друг на друга: валюта, направление, курс, резерв, реквизиты, KYC/AML, кеш, CRON | "Почему включил настройку, а ничего не изменилось?", "Что еще проверить?", "Почему направление не видно?" |
| Стартовые резервы и валюты | Расчет стартового резерва, популярные криптовалюты, первые направления и лимиты | "Сколько резерва нужно?", "Какие crypto-валюты добавить?", "Можно стартовать с 1000 USDT?" |
| Защита FastPanel | Безопасность панели сервера, доступы, IP-ограничение, backup, временные пользователи | "Как защитить FastPanel?", "Можно дать доступ поддержке?", "Что закрыть на сервере?" |
| Техническая настройка сервера | FastPanel, Nginx, PHP 8.4, база, Redis, SSL, права файлов, frontend build | "Как настроить Nginx?", "Что нужно для PHP 8.4?", "Что должен сделать серверщик?" |
| Nginx для Angular SSR | Frontend-домен, Angular SSR, `/backend-api`, backend upstream, `/ng-custom/`, static aliases | "Почему API дает CORS?", "Что писать в API_URL?", "Почему backend-api открывает сайт?" |
| CRON, очереди и Horizon | Scheduler, Redis queues, Horizon, фоновые задачи курсов, заявок, выплат, уведомлений и аналитики | "Почему процесс не запускается?", "Почему очередь стоит?", "Что должно работать на сервере?" |
| Обновления и сопровождение | Ошибки после обновления, новая версия, резервная копия, процессы, что прислать специалисту | "Где скачать новую версию?", "После обновления ошибка", "Не запускается процесс" |
| Релизы и новости | Подключение страницы релизов к GitBook AI и ответы по конкретным версиям | "Что нового в 11.1.5?", "Какой последний релиз?", "Что изменилось в версии?" |
| Реальные вопросы клиентов | Частые формулировки из поддержки по 2FA, курсам, BestChange, AML, бонусам и партнерам | "Почему парсер не показывает курс?", "Почему не начислился кешбек?", "Почему реферал не закрепился?" |

## Что AI должен уметь делать по каждой области

| Навык | Описание | Пример |
| --- | --- | --- |
| Найти раздел | Подсказать, куда зайти в админке | "Откройте Заявки — Список" |
| Объяснить поле | Сказать, что поле меняет для клиента или оператора | "Статус включает или скрывает валюту" |
| Объяснить галочку | Показать последствия включения | "Если включить автоудаление, неоплаченные заявки могут закрываться автоматически" |
| Дать диагностику | Предложить порядок проверки | "Проверьте валюту, направление, курс, резерв, кеш" |
| Предупредить о риске | Отметить, где действие может повлиять на деньги или безопасность | "Не повторяйте выплату без проверки первой попытки" |
| Попросить уточнение | Задать один полезный вопрос, если данных мало | "Заявка не видна в Live или во всех заявках?" |
| Направить к ручной проверке | Не придумывать ответ там, где нужен ответственный сотрудник | "Высокий AML-риск требует решения по регламенту" |

## Новые слои знаний для AI-чата

| Слой знаний | Для чего нужен | Когда использовать |
| --- | --- | --- |
| База вопросов для AI-чата | Быстрые ответы на частые вопросы | Когда пользователь задает простой вопрос по разделу |
| Расширенная база вопросов и ответов | Большой набор вопросов по всем основным разделам админки | Когда нужен более точный ответ с проверками и предупреждениями |
| Сценарии ответов AI-чата | Готовые playbook-сценарии диагностики | Когда пользователь пишет “не работает”, “зависло”, “не пришло” |
| Маршрутизатор задач | Карта “куда идти и что проверить первым” | Когда вопрос короткий и непонятно, какой раздел нужен |
| Полная карта админ-панели | Единая карта реальных разделов админки с назначением и проверками | Когда пользователь спрашивает, где находится настройка или как устроена админка целиком |
| Чек-листы настройки | Пошаговые инструкции настройки | Когда пользователь спрашивает “как правильно настроить” |
| Полная инструкция по формулам | Подробная работа с формулами, коэффициентами и парами | Когда вопрос касается курсов, маржи, fallback, inverse или BestChange |
| Комиссии, пересчет и ограничения направления | Подробный справочник по пяти вкладкам направления: прибыль, доп. и ПС, от суммы обмена, пересчет, ограничения и проверки | Когда вопрос касается итоговой суммы, комиссии ПС, диапазона суммы, FIX/FLOATING, стран, IP, новичков или лимитов заявок |
| После покупки и запуск | Порядок подготовки после покупки продукта | Когда пользователь спрашивает про сервер, домен, доступы, установку или ответственность сторон |
| Типовые ошибки после обновления | Безопасная диагностика после обновления без кода и внутренних путей | Когда пользователь пишет, что после обновления появилась ошибка или остановился процесс |
| Запуск обменника по шагам | Полный сценарий настройки обменника от нуля до тестовой заявки | Когда владелец хочет запустить обменник без программиста |
| Частые вопросы клиентов из Telegram | Реальные частые вопросы и готовые безопасные ответы | Когда вопрос звучит как короткое сообщение из поддержки |
| Стартовые резервы и популярные валюты | Практическая инструкция по резервам, валютам и лимитам запуска | Когда пользователь спрашивает, сколько денег держать в резерве и какие пары запускать |
| Защита FastPanel и сервера | Безопасная подготовка панели сервера | Когда вопрос касается серверной панели, доступа специалисту, backup или ограничения входа |
| Техническая настройка сервера | Runbook для технической команды по FastPanel, Nginx, PHP 8.4, базе, Redis и правам | Когда вопрос требует серверной настройки, а не только админки |
| Nginx для Angular SSR в FastPanel | Runbook по frontend-домену, SSR upstream, `/backend-api` и proxy-ошибкам | Когда вопрос про production frontend, CORS, `API_URL`, `/ng-custom/` или Nginx location |
| CRON, очереди и Horizon | Подробный слой по фоновым процессам | Когда вопрос про остановившиеся курсы, письма, выплаты, архив или аналитику |
| Релизы и новости для AI-чата | Правила подключения и использования публичной страницы релизов | Когда пользователь спрашивает про новую версию или изменения в релизе |

## Частые формулировки, которые должен понимать AI

| Как может спросить пользователь | Что это обычно означает |
| --- | --- |
| "Пара не работает" | Проверить направление обмена, курс, резерв, лимиты и статус валют |
| "Не видно валюту" | Проверить статус валюты, фильтр, направление и кеш |
| "Не приходит заявка" | Проверить создание заявки, список, фильтры, Live и уведомления |
| "Не выдает карту" | Проверить реквизиты, мерчант, лимиты и правила выдачи |
| "Курс кривой" | Проверить источник, формулу, BestChange, файл, кеш |
| "Оплата не подтянулась" | Проверить платежный журнал, сумму, реквизит и статус заявки |
| "Выплата упала" | Проверить автовыплату, провайдера, лимиты, реквизит и очередь |
| "Не работает бот" | Проверить Telegram Bot, канал, права, шаблон и тест |
| "Не приходит почта" | Проверить email, шаблон, событие, очередь и журнал |
| "Клиент не проходит проверку" | Проверить KYC, верификацию карты, AML или черный список |
| "Не применился бонус" | Проверить партнерскую программу, статус заявки, HOLD и журнал |
| "Не работает промокод" | Проверить статус, дату, лимит, направление и пользователя |
| "Забыли 2FA" | Проверить владельца доступа, журнал входов и процедуру безопасного восстановления |
| "Где скачать новую версию" | Направить к официальному источнику обновлений и предупредить про резервную копию |
| "Массово поменять проценты" | Проверить фильтр направлений, выбранное поле процента и тестовые пары |
| "Поставить комиссию ПС" | Открыть направление -> Доп. и ПС, выбрать сторону Отдаю/Получаю, указать процент/фикс/минимум и проверить тестовую сумму |
| "Комиссия зависит от суммы" | Открыть направление -> От суммы обмена, проверить диапазоны без пересечений и тестировать границы |
| "Включить плавающий курс" | Открыть направление -> Пересчет заявок, включить Фиксированный/Плавающий, задать описание, статусы, интервал и пороги |
| "Закрыть направление для страны/IP/новичка" | Открыть направление -> Ограничения и проверки, настроить страны, IP, KYC, лимит новичка или лимиты заявок |
| "Файл курсов не работает" | Проверить формат файла, дату обновления, группу курсов и выбранный источник в направлении |
| "BestChange не показывает" | Проверить пару, режим расчета, фильтры, исключения и дату обновления |
| "XML пустой" | Проверить статус направлений, включение экспорта, курс, резерв и лимиты |
| "После обновления ошибка" | Проверить версию, резервную копию, где видна ошибка, процессы и журналы |
| "Можно без домена" | Объяснить, что без домена возможна только временная подготовка, а не полноценный запуск |
| "CAPTCHA не проходит" | Проверить ключи сервиса, домен, режим защиты и жалобы клиентов |
| "Реферал не закрепился" | Проверить источник регистрации, старую привязку, срок cookie и правила программы |
| "Сколько резерва нужно" | Рассчитать по средней заявке, количеству одновременных заявок и запасу 1.3-1.5 |
| "Какие монеты добавить" | Начать с USDT TRC20, USDT BEP20, BTC, ETH, TRX, USDC и добавлять TON/SOL по спросу |
| "Как защитить FastPanel" | Проверить пароль, IP-ограничение, SSL, лишние пользователи, backup и журналы входов |
| "Что нового в релизе" | Найти точный номер версии в подключенном источнике релизов и не придумывать изменения |

## Что требует ручного уточнения владельца

| Тема | Почему нужна ручная проверка |
| --- | --- |
| Допустимый AML-риск | Значение зависит от политики обменника |
| Причины отказа KYC | Формулировки должны совпадать с правилами обменника |
| Статусы заявок | Рабочий регламент может отличаться у разных команд |
| Лимиты операторов | Зависят от команды, смен и уровня доверия |
| Ручные выплаты и возвраты | Это финансовые операции с высоким риском |
| Партнерские условия | Должны совпадать с публичной офертой |
| Юридические тексты | Должны быть проверены владельцем или юристом |
| Серверные требования | Зависят от нагрузки, количества заявок, частоты обновления курсов и подключенных сервисов |
| Процедура восстановления 2FA | Должна совпадать с правилами безопасности владельца |
| Процесс обновления | Нужны реальный порядок обновлений, доступ к официальной версии и политика резервных копий |
| Стартовые резервы | Зависят от среднего чека, региона, платежных методов и рекламного потока |
| Список стартовых валют | Должен учитывать спрос клиентов и реальные резервы владельца |
| Источник релизов | Нужно проверить, что GitBook Website Connection индексирует актуальную страницу релизов |
