# iEXExchanger Admin AI Knowledge Base

Эта база знаний предназначена в первую очередь для GitBook AI Agent и Assistant. Ее задача — дать AI-чату безопасные и подробные знания по админ-панели iEXExchanger, чтобы пользователи могли задавать вопросы простым языком и получать точные ответы по настройкам, заявкам, курсам, платежам, KYC, AML, уведомлениям и диагностике.

Эта база может быть подключена к публичному GitBook Site, но по смыслу она не является рекламным разделом и не должна читаться как обычная публичная инструкция для конечных клиентов обменника. Это справочный слой для AI-агента, поддержки, владельца обменника, администраторов и менеджеров.

Главная цель — помочь владельцам обменника, администраторам, операторам и поддержке получать понятные ответы по работе админки: где находится настройка, что она означает, когда ее менять, какие ошибки возможны и что делать в типовой ситуации.

Документация написана простым языком. Она объясняет практическое назначение разделов, полей, галочек, статусов и действий администратора. Здесь не должно быть исходного кода, внутренних путей проекта, секретов, токенов, паролей, приватных адресов, реальных данных клиентов или реквизитов.

## Как обновлять документацию

Обновляйте базу знаний каждый раз, когда меняется админка, форма, статус, настройка, уведомление, платежный провайдер или рабочий регламент поддержки.

| Ситуация | Что обновить | Как проверить |
| --- | --- | --- |
| В админке появился новый пункт меню | Добавить страницу или подраздел в подходящую папку | Проверить название раздела, назначение и права доступа |
| В форме появилось новое поле или галочка | Добавить строку в таблицу настроек | Объяснить, что поле меняет для сайта, заявки или оператора |
| Изменился статус заявки, KYC или AML | Обновить таблицу статусов и частых ошибок | Проверить, как статус видит оператор и что можно сказать клиенту |
| Добавлен новый платежный провайдер | Обновить раздел платежных систем | Не добавлять ключи, токены, приватные адреса и реальные реквизиты |
| Изменились фоновые процессы | Обновить раздел CRON, очередей и диагностики | Проверить, что написано, какие симптомы видит администратор |

## Как писать правильно

| Ситуация | Правильно | Неправильно |
| --- | --- | --- |
| Нужно объяснить галочку | "Если включить настройку, валюта станет доступна клиенту на сайте. Перед включением проверьте резерв, лимиты и реквизиты." | "Поставьте значение Да в техническом поле." |
| Нужно описать поле курса | "Укажите источник курса и проверьте итоговый расчет на примере пары BTC - USDT." | "Используется внутренний обработчик курса." |
| Нужно описать ошибку | "Если заявка зависла в ожидании оплаты, проверьте статус оплаты, журнал платежной системы и сумму поступления." | "Проверьте метод, маршрут или файл." |
| Нужно дать пример | "Например, для USDT TRC20 укажите сеть TRC20, минимальную сумму и отдельный резерв." | "Например, настройте как в коде." |
| Нужно предупредить об опасной настройке | "Не отключайте проверку реквизитов без теста, иначе клиенты смогут отправлять заявки с ошибочными данными." | "Можно выключить, если мешает." |

## Что запрещено добавлять

| Запрещенные данные | Почему нельзя добавлять |
| --- | --- |
| API-ключи, токены, пароли, секретные коды | Это приватные данные проекта и провайдеров |
| Реальные клиентские данные, заявки, e-mail, телефоны, документы | Это персональные данные клиентов |
| Реальные реквизиты, кошельки, карты, адреса выплат | Это платежная информация |
| Приватные адреса админки, серверов, журналов и хранилищ | Это повышает риск несанкционированного доступа |
| Внутренние пути проекта, названия файлов, классов, методов | Эта база должна объяснять работу админки, а не устройство проекта |

## Проверка качества

Перед публикацией новой страницы проверьте:

| Проверка | Что должно быть |
| --- | --- |
| Простота языка | Текст понятен владельцу обменника без программиста |
| Практическая польза | Есть ситуации, когда открывать раздел, и пошаговые действия |
| Поля и галочки | Каждое важное поле объяснено по смыслу и последствиям |
| Ошибки | Есть причина и действие, которое может выполнить администратор |
| Предупреждения | Отмечены настройки, влияющие на заявки, курсы, резервы, уведомления и безопасность |
| Безопасность | Нет секретов, приватных данных, внутренних путей и исходного кода |
