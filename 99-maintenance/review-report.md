# Краткий отчет по базе знаний админ-панели

## Что создано

Создана публично безопасная база знаний для GitBook AI по админ-панели. База предназначена для владельца обменника, менеджеров, поддержки и клиентов, которые задают вопросы простым языком по работе админки.

## Непонятные места или места для уточнения

| Место | Что нужно уточнить |
|---|---|
| Индивидуальные платежные провайдеры | У каждого шлюза могут быть свои поля подключения, которые зависят от провайдера и не должны раскрывать секреты |
| Реальные регламенты операторов | Доступные действия есть в админке, но бизнес-правила “когда нажимать” должен подтвердить владелец |
| Режимы автобанов и антиспама | Значения есть в настройках, но допустимая строгость зависит от потока клиентов |
| Статусы KYC/AML | Технические статусы есть, но текст для клиента лучше согласовать с compliance-политикой |
| Сроки хранения логов | Нужны бизнес- и юридические требования владельца |
| Чат заявки | Реальные правила использования чата должны совпадать с регламентом поддержки |
| Экспорт данных | Точный список допустимых выгрузок зависит от прав роли и политики хранения данных |
| Ответы AI-чата | Для точных ответов по спорным финансовым ситуациям нужны реальные регламенты владельца обменника |
| Восстановление 2FA | Нужно утвердить процедуру проверки владельца и список ответственных лиц |
| Официальные обновления | Нужно подтвердить, какой канал считается правильным для скачивания новой версии |
| Партнерские начисления по API | Нужно подтвердить, какие данные можно отдавать партнеру и какие права API считаются безопасными |
