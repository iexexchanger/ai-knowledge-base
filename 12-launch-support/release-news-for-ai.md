# Релизы и новости для AI-чата

## Для чего нужен раздел

Этот раздел объясняет, как GitBook AI должен отвечать на вопросы о новых версиях iEXExchanger: “что нового в версии 11.1.5”, “какие изменения были в релизе”, “где скачать новую версию”, “какой последний релиз”.

Главный источник новостей о релизах — публичная страница:

`https://iexexchanger.com/news/relizy`

## Важно понимать

GitBook AI не должен придумывать изменения релиза. Если пользователь спрашивает про конкретную версию, ответ должен быть основан на странице релиза или на проиндексированной записи GitBook Website Connection.

Если источник релизов не подключен к GitBook Assistant, AI может знать только то, что уже вручную добавлено в базу знаний. Поэтому для актуальных ответов нужно подключить страницу релизов как внешний источник.

## Как подключить релизы к GitBook AI

| Шаг | Что сделать | Зачем |
| --- | --- | --- |
| 1 | Открыть GitBook и перейти в Connections | Там подключаются внешние источники знаний |
| 2 | Выбрать Website | Это источник для страниц сайта |
| 3 | Выбрать Answer user questions | Чтобы Assistant использовал записи для ответов пользователям |
| 4 | Указать страницу релизов `https://iexexchanger.com/news/relizy` | Это главный список релизов |
| 5 | Включить Expose in Search | Чтобы записи попадали в поиск |
| 6 | Включить Expose in Assistant | Чтобы AI-чату можно было отвечать по релизам |
| 7 | Дождаться индексации | GitBook должен прочитать страницы |
| 8 | Проверить вопросом “Что нового в iEXExchanger 11.1.5?” | Так можно убедиться, что источник работает |

Предупреждение: если включить Answer user questions, записи из источника станут доступны пользователям документации через Assistant. Страница релизов публичная, поэтому это нормальный источник для публичного AI-чата.

## Как AI должен отвечать про релизы

| Вопрос пользователя | Как отвечать |
| --- | --- |
| Что нового в версии 11.1.5? | Найти запись релиза 11.1.5 и кратко перечислить изменения простым языком |
| Что изменилось в последнем релизе? | Найти самый свежий релиз в источнике и дать краткое резюме |
| Где скачать новую версию? | Напомнить, что скачивать нужно только из официального источника обновлений, а не из случайных архивов |
| Нужно ли обновляться? | Объяснить изменения релиза и посоветовать сначала сделать резервную копию и проверить совместимость |
| После обновления появилась ошибка | Направить в раздел типовых ошибок после обновления и попросить версию до обновления, версию после обновления и место ошибки |

## Что нельзя делать

| Нельзя | Почему |
| --- | --- |
| Придумывать список изменений | Можно ввести клиента в заблуждение |
| Говорить “в последней версии исправлено”, если источник не найден | Нужна проверка по релизу |
| Давать ссылку на неофициальный архив | Это риск взлома и повреждения проекта |
| Советовать обновлять рабочий проект без backup | При ошибке будет сложно восстановиться |
| Обещать, что обновление решит конкретную проблему без проверки | Причина ошибки может быть не в версии |

## Как отвечать, если релиз не найден

Если пользователь спрашивает про версию, которой нет в источнике:

1. Сказать, что в подключенном источнике эта версия не найдена.
2. Предложить проверить официальную страницу релизов.
3. Попросить точный номер версии.
4. Не придумывать изменения.
5. Если вопрос связан с ошибкой после обновления, перейти к диагностике обновления.

Пример ответа:

“Я не нашел релиз с таким номером в подключенной базе. Проверьте точный номер версии на странице релизов. Если ошибка появилась после обновления, пришлите версию до обновления, версию после обновления, где видна ошибка и есть ли резервная копия.”

## Как обновлять эту страницу

| Когда | Что сделать |
| --- | --- |
| Вышел новый релиз | Проверить, что страница релизов проиндексирована GitBook Website Connection |
| GitBook AI не отвечает про новый релиз | Перезапустить или обновить индексацию источника Website |
| Изменился адрес релизов | Обновить источник в GitBook Connections и эту страницу |
| В релизе есть важное предупреждение | Добавить краткое пояснение в базу знаний, если это влияет на запуск или обновление |
