# Пользователи, роли и доступы

Этот справочник описывает разделы клиентов, администраторов, ролей, блокировок, доп. полей, скидок, уровней клиентов и бонусный центр.

## Список пользователей

| Поле / действие | Что означает | Как использовать | Рекомендация |
|---|---|---|---|
| ID пользователя | Внутренний номер аккаунта | Ищите по нему при точной диагностике | Не путайте с ID заявки |
| Email | Основной контакт и логин | Ищите по жалобе клиента | Не публикуйте email в открытых каналах |
| Имя | Имя пользователя | Проверяйте при ручной идентификации | Может быть автосгенерированным |
| Телефон | Контакт, если включён | Используйте только по регламенту | Чувствительные данные |
| Статус | Активен или заблокирован | Проверяйте, если клиент не может войти | Не блокируйте без причины |
| Роль | Права администратора или тип пользователя | Для админов определяет доступ | Обычному клиенту не давайте админские роли |
| Дата регистрации | Когда создан аккаунт | Важно для антиспама и истории | Сравнивайте с заявками |
| Последний вход | Когда пользователь был активен | При вопросах доступа | Может зависеть от сессий |
| Фильтр по IP/email | Поиск подозрительных аккаунтов | Используйте при спаме | Учитывайте прокси и NAT |
| Открыть пользователя | Перейти в карточку | Для балансов, заявок, банов, сессий | Смотрите логи перед изменениями |

## Основные поля пользователя

| Поле / настройка | Что означает | Как правильно заполнять | Рекомендация |
|---|---|---|---|
| Email | Логин и контакт | Указывайте актуальный адрес | При смене email проверьте подтверждение |

| Роль | Права пользователя в админке | Только для админов и сотрудников | Ошибка может дать полный доступ |
| Комментарий | Внутренняя заметка | Пишите фактами | Не используйте грубые оценки |

|---|---|---|---|

## Бан пользователя

| Поле / настройка | Что означает | Как правильно заполнять | Рекомендация |
|---|---|---|---|
| Включить бан | Заблокировать пользователя | Включайте только по причине | Не баньте без записи причины |
| Причина | Почему заблокирован | Спам, мошенничество, нарушение правил | Формулируйте фактами |
| Срок | До какого времени действует бан | Временный или постоянный | Для спорных случаев лучше временный |
| История банов | Прошлые блокировки | Смотрите перед решением | Повторные нарушения усиливают меры |
| Снять бан | Разрешить доступ | После проверки и решения | Оставьте комментарий |

## Активные сессии и 2FA

| Поле / действие | Что означает | Как использовать | Предупреждение |
|---|---|---|---|
| Активная сессия | Устройство, где пользователь вошёл | Проверяйте IP, дату, устройство | IP может быть неточным при плохом прокси |
| Удалить сессии | Завершить входы пользователя | При подозрении на взлом | Пользователь выйдет из аккаунта |
| Google 2FA включена | Пользователь использует 2FA | Проверяйте при жалобе на вход | Не сбрасывайте без идентификации |
| Сбросить Google 2FA | Отключить старую привязку | Только после проверки личности | Опасное действие |
| Включить/обновить 2FA | Настроить защиту | Для админов обязательно | Сохраните резервный процесс восстановления |

## Роли

| Поле / настройка | Что означает | Как правильно заполнять | Рекомендация |
|---|---|---|---|
| Название роли | Имя набора прав | “Оператор”, “Старший оператор”, “Поддержка”, “Владелец” | Название должно отражать обязанности |
| Описание | Для чего роль | Короткое пояснение | Помогает не путать |
| Статус | Роль активна | Включайте используемые роли | Выключение роли может закрыть доступ |
| Права меню | Какие разделы доступны | Выдавайте минимально нужные | Не давайте всё подряд |
| Права действий | Можно ли сохранять, удалять, выполнять | Разделяйте просмотр и выполнение | Самые опасные — выплаты, роли, настройки |
| Заявки | Доступ к списку и карточке | Операторам нужен просмотр | Выполнение — отдельная ответственность |
| Выполнение заявок | Подтверждение, отклонение, выплата | Только обученным сотрудникам | Ошибка влияет на деньги |
| Пользователи | Просмотр и изменение клиентов | Для поддержки и старших операторов | Содержит персональные данные |
| Роли | Управление доступами | Только владельцу | Это право может дать полный контроль |
| Настройки | Системные параметры | Только владельцу и техадмину | Ошибка может сломать сайт |
| Мерчанты / автовыплаты | Платёжные интеграции | Только фин. ответственным | Ошибка влияет на оплату |
| KYC / AML | Проверки документов и рисков | Для compliance-роли | Очень чувствительные данные |
| Экспорт | Выгрузка данных | Только доверенным | Файлы могут содержать персональные данные |
| Логи | История действий | Для контроля и диагностики | Не всем сотрудникам |

| Поле / настройка | Что означает | Как правильно заполнять | Рекомендация |
|---|---|---|---|
| Кампания | Бонусная акция | Назовите по смыслу | Например “Бонус за первую заявку” |

## Типовые ошибки

| Ситуация | Причина | Что сделать |
|---|---|---|
| Клиент не может войти | Бан, неверный пароль, 2FA, сессия, email не подтверждён | Проверить пользователя, сессии, логи входа |
| Админ не видит раздел | Нет права в роли | Проверить роль и permission |
| Клиент не видит кошельки | Валюты кошельков не выбраны или выключены | Проверить настройки пользователей и валюты |
| Скидка не применяется | Программа выключена, условия не выполнены, нужен пересчёт | Проверить скидку и историю пользователя |
| Уровень клиента не изменился | Не запущен пересчёт или не выполнены условия | Запустить пересчёт и проверить KYC |
| Бонус не начислился | Кампания неактивна или условие не выполнено | Проверить бонусный центр журналы/метрики |
| Заблокировали не того клиента | Неверный IP/email или общий NAT | Проверить логи и снять блок при ошибке |

## Безопасные рекомендации

1. Не выдавайте права “на всякий случай”.
2. Для операторов разделяйте просмотр заявки и выполнение заявки.
3. Доступ к ролям, настройкам, API, мерчантам и автовыплатам оставляйте владельцу или техадмину.
4. Перед баном клиента проверьте заявки, IP, комментарии и логи.
5. Перед изменением баланса оставляйте понятную причину.
6. Не копируйте персональные данные клиентов в публичные каналы.
