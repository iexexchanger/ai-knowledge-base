# Подробный справочник настроек

Этот файл описывает реальные настройки админки простым языком. Если пользователь спрашивает про конкретную галочку или поле в разделе “Настройки”, AI должна искать здесь.

## Основные настройки сайта

| Поле / настройка | Что означает | Как правильно заполнять | Рекомендация |
|---|---|---|---|
| Название сайта | Основное имя обменника | Укажите бренд или название проекта | Должно совпадать с логотипом и письмами |
| Краткое название | Короткое описание бренда | 2–5 слов | Используйте в шапке или малых блоках |
| Description сайта | SEO-описание | 1–2 предложения до 200 символов | Не пишите длинный рекламный текст |
| Часовой пояс | В каком времени показывать даты | Выберите часовой пояс фактической работы | Влияет на заявки, логи, CRON, отчёты |
| Текущий язык | Основной язык админки и сайта | Выберите рабочий язык | Если команда русскоязычная, оставляйте русский |
| Мультиязычность | Какие языки доступны на сайте | Указывайте только реально заполненные языки | Пустые переводы ухудшают сайт |
| Отключить смену языка | Запретить клиенту менять язык | Включайте, если сайт должен работать на одном языке | Если включено, клиент не увидит переключатель |
| Вид отображения счёта | Как скрывать реквизиты пользователя | Выберите полный показ или маскирование | Маскирование снижает риск утечки реквизитов |
| Направление по умолчанию | Какой режим выбора направления открывается первым | Оставьте рекомендованный режим или выберите из настроек направления | Влияет на первый экран обмена |
| Внутренний счёт | Включает внутренний баланс клиента | Включайте только если используете личный кабинет и балансы | Нужна проверка балансов и валют |
| Аккордеон контента | Сворачивать длинный контент на страницах | Включайте для длинных страниц | Проверьте, что важные условия видны |
| Кнопки “Читать далее” | Показывать кнопку для длинных текстов | Включайте при длинных описаниях | Не скрывайте критичные комиссии и правила |
| Количество строк “Читать далее” | После скольких строк скрывать текст | Обычно 5–8 | Слишком мало строк раздражает клиента |

## Баланс системы

|---|---|---|---|

## Настройки обмена и заявок

| Поле / настройка | Что означает | Как правильно заполнять | Рекомендация |
|---|---|---|---|
| Публичный ID заявки | Какой номер видит клиент | Стандартный ID проще, алгоритмический сложнее угадать | Для приватности лучше алгоритмический ID |
| Отображение курса или даты | Показывать клиенту курс заявки или только дату | Включайте, если клиенту важно видеть курс фиксации | При спорах курс должен совпадать с заявкой |
| Отключить поле email | Скрыть email для новых клиентов | Включайте только если бизнес допускает технический email | Может усложнить связь с клиентом |
| Стратегия ошибок мерчанта | Что делать, если мерчант не выдал реквизиты | Без повторов, повторять, использовать альтернативный мерчант | Для стабильности лучше иметь альтернативу, если она настроена |
| Способ пересчёта заявки | Когда пересчитывать заявку | При смене статуса, по CRON или оба варианта | Для плавающих курсов лучше согласовать с регламентом |
| Интервал обновления курсов на странице | Как часто публичный сайт обновляет курсы | Не ставьте слишком часто | Рекомендуется не меньше 15 секунд |
| Максимум нажатий “Оплатить и завершить” | Сколько раз клиент может нажать кнопку оплаты | `0` отключает ограничение | Ограничение помогает от повторных запросов |
| Пересчёт после истечения времени | Можно ли пересчитать заявку после просрочки | Включайте для мерчантов с поздним подтверждением | Обязательно объясните клиенту условия |
| Инструкция мерчанта пустая | Что показать, если у мерчанта нет инструкции | Ничего или инструкцию направления | Лучше иметь fallback из направления |
| Отклонять неполную оплату через мерчант | Что делать при неполной оплате | Обычно включают, если мерчант строго проверяет сумму | Может отклонить реальную частичную оплату |
| Автобан за мошеннические заявки | Банить клиента при scam-отклонении | Включайте после согласования с командой | Нужны корректные причины отклонения |
| Режим автобана | Строгость наказаний | Мягкий, стандартный, жёсткий | Жёсткий используйте только при атаках |
| Блокировать за спам-заявки | Автоматически ограничивать спамеров | Включайте при частых пустых заявках | Смотрите логи, чтобы не блокировать обычных клиентов |
| Добавлять IP/email в черный список | Автоматически заносить данные в чёрный список | Включайте только при понятном регламенте | Ошибка оператора может заблокировать нормального клиента |
| Запись данных заявки в файл | Дублировать данные заявок в файл | Включайте только если это нужно для интеграции или резервной отчётности | Файл может содержать чувствительные данные |
| Доступ к чеку заявки | Кто может открыть чек | Для приватности выбирайте владельца или временный доступ | Слишком строгий режим может вызвать жалобы |
| Максимум заявок на клиента | Ограничение количества заявок | `0` обычно означает без ограничения | Полезно против спама |
| Отображаемые статусы в ЛК | Какие заявки клиент видит в личном кабинете | Выбирайте статусы, которые нужны клиенту | Не показывайте внутренние статусы без смысла |
| Время на оплату заявки | Сколько времени клиенту даётся оплатить | Укажите реалистичное время по платёжным методам | Для crypto нужно учитывать подтверждения сети |
| Автоархив | Автоматически переносить старые заявки в архив | Включайте для больших проектов | Не архивируйте слишком рано |
| Через сколько дней архивировать | Возраст заявки для архива | Задайте срок по регламенту | Для споров храните рабочий доступ достаточно долго |
| Статусы для архива | Какие заявки архивировать | Выбирайте финальные статусы | Не архивируйте активные заявки |

## Антиспам заявок

| Поле / настройка | Что означает | Как правильно заполнять | Рекомендация |
|---|---|---|---|
| Включить антиспам | Система ограничивает подозрительные заявки | Сначала включайте мягко | Проверяйте жалобы после включения |

| Разрешённое количество быстрых заявок | Сколько заявок можно создать быстро | Обычно 3–5 | Слишком низко мешает клиентам |
| Период неоплаченных заявок | За сколько часов искать неоплаченные | Обычно 24 часа | Помогает не занимать резервы |
| Допустимое количество неоплаченных | Сколько неоплаченных заявок разрешить | Обычно 2–3 | Для B2B-клиентов может быть больше |
| Период проверки личности | Окно анализа похожих пользователей | Обычно 72 часа | Нужна осторожность для офисных IP |

| Балл репутации | Порог подозрительности | Обычно 70 | Не путать с AML risk score |
| Блокировка личности | На сколько часов ограничить подозрительную связку | Обычно 24 часа | При ошибке клиент не сможет создать заявку |

## Безопасность

| Поле / настройка | Что означает | Как правильно заполнять | Рекомендация |
|---|---|---|---|
| Google Authenticator | Код 2FA при входе | Включайте после настройки ключевых аккаунтов | Обязательно для владельца |
| Доверенные прокси | Откуда брать реальный IP | Выберите файл только при реальном прокси | Неверно настроенный прокси ломает IP-логи |
| Время жизни сессии | Сколько минут действует вход | 60–240 минут | Для операторов не ставьте слишком мало |
| Максимум неверных входов | Лимит ошибок пароля | 5–10 | Защищает от перебора |
| Блокировка после ошибок | Пауза после неверных входов | От 10 минут | При атаке увеличьте |
| Контроль изменения IP | Сброс сессии при смене IP | Средний режим безопасен, высокий строже | Для мобильных операторов высокий режим может мешать |
| Ограничить админку по IP | Пускать только с разрешённых IP | Включайте только со стабильными IP | Можно закрыть доступ всем |
| Разрешённые IP | Список IP/CIDR | Добавьте все офисные и резервные IP | Храните список приватно |
| Сбрасывать ключ при каждом входе | Старые ключи становятся недействительными | Включайте для строгой политики | Может чаще завершать сессии |
| Показывать активные сессии | Пользователь видит устройства | Включайте для контроля | Требует корректного IP |
| CAPTCHA для входа, регистрации, заявки, админки и сброса пароля | Где включена CAPTCHA | Включайте по риску спама | Проверяйте нужную форму после включения |
| CAPTCHA site key | Публичный ключ | Вставьте из сервиса CAPTCHA | Не путать с secret |
| CAPTCHA secret | Секретный ключ | Хранить только в настройках | Нельзя добавлять в документацию |

## Какие настройки зависят от других разделов

| Настройка | Не заработает корректно без | Что проверить |
|---|---|---|
| Включить внутренний счет | Валюты, баланс системы, личный кабинет, права пользователя | Есть ли основная валюта баланса и понятный сценарий пополнения |
| Скрывать валюты без резерва | Валюты, направления, резервы, кеш главной | Резерв заполнен и не равен нулю |
| Показ популярных направлений | Направления, статистика заявок, кеш | Есть реальные успешные обмены или ручной порядок |
| Включить чат заявки | Статусы заявок, права операторов, уведомления | Чат разрешен для нужных статусов |
| Telegram-уведомления чата | Telegram-канал, бот, события, очередь | Бот привязан и имеет права отправки |
| Email-уведомления чата | Почтовая служба, события писем, очередь | Тестовое письмо проходит |
| Пересчет заявки | Курсы, настройка направления, CRON, статусы | Понятно, когда пересчет разрешен по регламенту |
| Автоархив заявок | Статусы, возраст заявки, scheduler | Выбраны только финальные статусы |
| Антиспам заявок | Логи заявок, лимиты, идентификаторы клиента | Начинать с мягкого режима и смотреть жалобы |
| CAPTCHA при создании заявки | Ключи CAPTCHA, домен, публичная форма | Проверить реальную форму после включения |
| Очистка логов | Регламент хранения, поддержка, финансы | Не удалять нужную историю платежей и споров |
| Кеш публичных блоков | Контент, курсы, резервы, очистка кеша | После изменения важного блока проверить сайт |
| AI в онлайн-чате | База знаний, правила ответов, онлайн-чат | AI не должен отвечать по секретам и спорным платежам |

## Готовые ответы по частым вопросам настроек

| Вопрос пользователя | Короткий ответ AI |
|---|---|
| "Где поменять название сайта?" | Откройте Настройки - Основное. Заполните название и краткое название для нужных языков, затем проверьте сайт и письма. |
| "Почему время в заявке неверное?" | Проверьте Настройки - Основное - Часовой пояс. После изменения смотрите новые заявки и логи, старые записи могли быть созданы в прежней зоне. |
| "Почему клиент видит старый текст?" | Проверьте текст/контент на нужном языке и очистите кеш в Настройки - Оптимизация, если публичный блок кешируется. |
| "Почему не приходит письмо?" | Проверьте Настройки - E-mail: общий тумблер уведомлений, почтовый драйвер, отправителя, нужное событие и тестовую отправку. Если тест проходит, проверьте очередь. |
| "Почему Telegram не отправляет?" | Проверьте общий тумблер Telegram, токен бота, привязку канала, права бота, выбранные события и очередь. |
| "Почему направление не появилось после включения?" | Проверьте не только статус направления, но и валюты, курс, резерв, лимиты, реквизиты или мерчант, KYC/AML и кеш. |
| "Почему курс не обновляется?" | Проверьте тип курса в направлении, источник или формулу, затем Настройки - CRON и последний успешный запуск фоновых задач. |
| "Можно ли включить автовыплаты сразу?" | Лучше сначала проверить ручную обработку и тестовую выплату на безопасной сумме. Автовыплаты зависят от провайдера, лимитов, резерва, статусов и очереди. |
| "Как защитить админку?" | Включите 2FA, настройте лимиты неверных входов, CAPTCHA, роли сотрудников и IP-ограничение только если есть стабильные доверенные IP. |
| "Что делать, если включил IP-ограничение и потерял доступ?" | Это серверная ситуация. Нужен ответственный специалист с доступом к серверу или панели. Нельзя советовать обходить защиту без проверки владельца. |
