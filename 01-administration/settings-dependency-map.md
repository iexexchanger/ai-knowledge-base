# Карта связей настроек админки

## Для чего нужен раздел

Эта страница объясняет, как настройки админки связаны между собой. Она нужна GitBook AI, когда пользователь спрашивает не только "что означает поле", но и "почему это влияет на заявку", "что еще нужно проверить", "почему после включения настройки ничего не изменилось".

Настройки iEXExchanger нельзя рассматривать отдельно. Например, направление может быть включено, но не показываться из-за резерва, курса, лимита, KYC, AML, кеша или отсутствия реквизитов.

## Главный принцип

Если настройка влияет на клиента, всегда проверяйте всю цепочку:

1. Общие настройки сайта.
2. Валюта.
3. Направление.
4. Курс.
5. Лимиты.
6. Резерв.
7. Реквизиты, мерчант или автовыплата.
8. KYC, AML, CAPTCHA, антиспам.
9. Уведомления.
10. CRON, очередь, кеш и журналы.

## Связи по основным вопросам

| Вопрос пользователя | Главная настройка | Что ещё проверить | Почему связано |
| --- | --- | --- | --- |
| "Не видно валюту" | Валюты, статус валюты | Фильтры валют, резерв, направление, кеш, дизайн списка валют | Валюта может быть активна, но не иметь доступного направления или скрываться без резерва |
| "Не видно направление" | Карточка направления | Статус обеих валют, курс, min/max, резерв, реквизиты, KYC/AML, экспорт, кеш | Направление показывается только если вся цепочка доступности проходит проверки |
| "Курс неправильный" | Направление - Курсы | Источник, формула, BestChange, файл, коэффициенты, настройки точности, CRON | Итоговый курс часто собирается из нескольких источников и надбавок |
| "Сумма округляется странно" | Настройки - Расчеты | Точность валюты, формула, комиссии, отображение курса | Есть реальная точность расчета и отдельно формат отображения |
| "XML пустой" | Настройки - Экспорт курсов | Экспорт в направлении, статус направления, резерв, min/max, исключенные направления, ручной режим оператора | Файл экспорта фильтрует направления по своим правилам |
| "Клиент не получил реквизиты" | Реквизиты или мерчант | Валюта, направление, правила мерчанта, лимиты, платежная система, журнал запросов | Реквизит выдается только при совпадении условий |
| "Заявка не создается" | Настройки - Обмен | Направление, поля формы, лимиты, антиспам, CAPTCHA, KYC/AML, черный список | Создание заявки блокируют и бизнес-правила, и защита |
| "Заявка не меняет статус" | Карточка заявки | Платежный журнал, очередь, мерчант, сумма, callback, ручные действия оператора | Статус зависит от оплаты, провайдера и фоновой обработки |
| "Не пришло письмо" | Настройки - E-mail | Общий тумблер уведомлений, событие, шаблон, очередь, лог почты | Даже верный SMTP не отправит письмо, если событие выключено |
| "Telegram молчит" | Настройки - Telegram | Бот, канал, права бота, тип события, шаблон, общий тумблер Telegram, очередь | Telegram требует и рабочий бот, и включенное событие |
| "Чат заявки не виден" | Настройки - Онлайн чат | Статусы показа чата, статус заявки, права оператора, email/Telegram уведомления | Чат может быть включен, но скрыт для конкретных статусов |
| "Клиент заблокирован" | Пользователи, Черный список, Антиспам | IP, email, реквизит, scam-режим, лимиты профиля, AML/KYC | Блокировка может прийти из нескольких подсистем |
| "Админ не может войти" | Настройки - Безопасность | 2FA, IP-ограничение, CAPTCHA, лимит неверных входов, сессия, роль | Доступ зависит от защиты входа и прав пользователя |
| "Сайт показывает старые данные" | Настройки - Оптимизация | Кеш публичных блоков, cache driver, CRON, дата изменения, браузерный кеш | Кеш ускоряет сайт, но может задержать обновление данных |

## Секции системных настроек и что они контролируют

| Секция в админке | Технический mid | Контролирует | Когда смотреть |
| --- | --- | --- | --- |
| Основное | `main` | Название, SEO-описание, часовой пояс, языки, отображение реквизитов, поведение контента | При запуске сайта, смене языка, неправильном времени |
| Баланс системы | `balance` | Главную валюту внутреннего баланса и fallback-коды | Перед включением балансов, партнерки и аналитики |
| Обмен | `orders` | Публичный номер заявки, пересчет, время оплаты, архив, отображаемые статусы, scam-режим | Когда вопрос связан с созданием или жизненным циклом заявки |
| Антиспам заявок | `security_order_anti_spam` | Частые заявки, неоплаченные заявки, похожие устройства, временные блокировки | При пустых или массовых заявках |
| Безопасность | `security` | 2FA, trusted proxy, сессии, IP-ограничения, CAPTCHA | При входе в админку, спаме, защите форм |
| Пользователи | `users` | Авторегистрацию, домены email, сохранение реквизитов, очистку неактивных пользователей | Когда вопрос связан с регистрацией и кабинетом клиента |
| Расчеты | `calculator` | Точность сумм и автозаполнение суммы в форме обмена | При округлениях и поведении поля суммы |
| Курсы | `rates-settings` | Историю курсов и количество знаков отображения | При диагностике курса и просмотра в админке |
| Оформление | `interface-design` | Тему, блоки главной, резервы, фильтры, список валют, footer, баннеры | Когда вопрос про внешний вид сайта |
| Изображения | `interface-image` | Фон, favicon, логотип, brand icon, логотип email | Когда нужно заменить визуальные файлы |
| Текст | `interface-text` | Техперерыв, приветствие, заголовки, footer-текст | Когда нужно поменять публичные тексты |
| E-mail | `notifications-email` | SMTP/Resend, отправителя, поддержку, админские адреса, события писем | Когда письма не приходят или меняется почта |
| Broadcast | `notification-broadcasting` | Realtime-уведомления на сайте и сообщение успешного обмена | Когда клиент не видит всплывающее уведомление |
| Онлайн чат | `online-chat` | Виджет чата, чат заявки, файлы, быстрые ответы, Jivo-сценарии | Когда нужна поддержка в заявке |
| Интеграции | `integrations` | Yandex Metrika, Google Analytics, Clarity, Bitcoin network load | При подключении аналитики |
| CRON | `others-cron` | Режимы обновления курсов, BestChange и файлов | Когда данные устаревают или задачи запускаются часто |
| GeoIP | `others-geo-ip` | Определение страны/города, кеш и аварийный readonly-режим | При региональных правилах и проблемах IP |
| Логи | `others-logs` | Автоочистку истории заявок, резервов, курсов, авторизации и экспорта | Когда база растет или нужны сроки хранения |
| Оптимизация | `optimization` | Кеш, быстрые payload сайта, кеш блоков, очистку кеша | При скорости сайта или старых данных |
| AI | `ai` | Ключ AI, AI в онлайн-чате, автоперевод | Когда включают GitBook/AI-сценарии и переводы |

## Настройки, которые требуют теста после сохранения

| Настройка | Тест |
| --- | --- |
| Часовой пояс | Создать тестовое событие или заявку и проверить время в админке |
| Языки сайта | Открыть публичный сайт на каждом включенном языке |
| Внутренний счет | Проверить кабинет клиента, баланс и операции |
| Публичный ID заявки | Создать тестовую заявку и сравнить номер в админке и у клиента |
| Пересчет заявки | Проверить тестовую заявку с плавающим курсом по регламенту |
| Время на оплату | Проверить страницу оплаты и поведение после истечения времени |
| Архив заявок | Сначала открыть preview или историю архивирования, потом включать автоматический режим |
| CAPTCHA | Проверить каждую форму, где она включена |
| IP-ограничение админки | Проверить резервный вход до выхода из текущей сессии |
| Email | Отправить тестовое письмо |
| Telegram | Проверить привязку канала и тестовое уведомление |
| Онлайн чат | Создать тестовую заявку и написать сообщение от клиента |
| CRON | Проверить последний успешный запуск после выбранного интервала |
| Кеш | Очистить кеш и убедиться, что сайт показывает новые данные |
| AI | Проверить тестовый ответ или автоперевод, не отправляя секреты в публичный чат |

## Что AI должен предупреждать

| Тема | Предупреждение |
| --- | --- |
| Баланс системы | Основную валюту баланса не меняют часто на рабочем проекте: это влияет на аналитику, бонусы и расчеты |
| Автовыплаты | Нельзя включать на большие суммы без теста и журнала первой выплаты |
| Пересчет заявки | Пересчет влияет на сумму клиента и может быть причиной споров |
| XML-экспорт | Изменение правил экспорта влияет на мониторинги и партнерские сервисы |
| IP-ограничение | Ошибка в списке IP может закрыть доступ в админку |
| CAPTCHA | Неверные ключи или домен могут заблокировать реальные заявки |
| Очистка логов | Слишком короткий срок хранения мешает расследованию платежей и споров |
| Кеш | Слишком длинный TTL может показывать клиентам старые данные |
| AI-ключ | Значение ключа нельзя писать в документацию, чат или публичный тикет |
