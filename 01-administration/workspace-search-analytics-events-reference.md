# Рабочий стол, поиск, аналитика и журнал событий

Эта страница объясняет первые разделы, которые администратор обычно открывает после входа: рабочий стол, глобальный поиск, аналитику и журналы. Они помогают не настраивать обменник вслепую: рабочий стол показывает текущее состояние, поиск быстро ведет к нужному месту, аналитика показывает результат за период, а журналы объясняют, что именно произошло.

## Что открыть первым

| Ситуация | Куда идти | Что смотреть |
|---|---|---|
| Начало смены оператора | Рабочий стол | Активные заявки, оплаченные заявки, ошибки выплат, зависшие процессы |
| Нужно быстро найти раздел или объект | Глобальный поиск в верхней панели | Раздел меню, заявка, пользователь, валюта, направление, мерчант, резерв |
| Нужно понять прибыль и оборот | Аналитика | Период, завершенные заявки, прибыль, направления, валюты, партнеры |
| Нужно понять причину ошибки | Журналы | Статусы заявки, AML, реквизиты, KYC, мерчанты, автовыплаты, партнерские события |
| Есть подозрение на доступ | Журналы пользователей и мульти-авторизации | IP, устройство, результат входа, активные сессии |

## Рабочий стол

Рабочий стол находится на главной странице админки. Это не форма настройки, а операционная сводка. Он помогает владельцу и оператору быстро понять, где сейчас есть работа или риск.

### Периоды

| Период | Что означает | Когда использовать |
|---|---|---|
| За все время | Общая картина по всей базе | Для грубой оценки масштаба обменника |
| Сегодня | Только текущий день | В начале и конце смены |
| 7 дней | Последняя неделя | Для контроля рекламы, нагрузки, новых проблем |
| 30 дней | Последний месяц | Для сравнения прибыли, заявок и партнеров |
| 12 месяцев | Годовая динамика, если доступна | Для долгой аналитики и сезонности |

### Основные карточки

| Блок | Что показывает | Как использовать |
|---|---|---|
| Операции | Все заявки за период и активные заявки сейчас | Если активных много, оператору нужно открыть список заявок |
| Ожидают оплаты | Заявки, где клиент еще должен оплатить | Обычно не требуют действия, но важны для нагрузки |
| Нужна обработка | Заявки, которые ждут оператора | Это рабочая очередь, ее проверяют первой |
| Оплачено | Клиент оплатил, дальше нужна проверка или выплата | Приоритетный блок для оператора |
| Заморожено | Заявки временно остановлены | Нужно открыть карточку и понять причину |
| Ошибки выплат | Автовыплата не прошла или требует внимания | Высокий риск, проверять журнал автовыплат и карточку заявки |
| Успешные заявки | Сколько заявок закрыто успешно за период | Используется для оценки результата |
| Прибыль USD | Прибыль по сохраненным profit-записям | Сравнивайте только с тем же периодом и статусами |
| Активные резервы | Сколько резервов включено в работу | Если мало активных резервов, часть направлений может быть недоступна |
| Пустые резервы | Резервы с нулем или отрицательной суммой | Нужно открыть "Резервы" и проверить валюты |

### Качество работы

| Показатель | Что означает | Как читать |
|---|---|---|
| Процент успешных | Доля заявок, закрытых успешно | Низкое значение может означать проблемы с курсом, реквизитами или оплатой |
| Отказы и ошибки | Доля отказов и ошибок выплат | Рост показателя требует проверки заявок и журналов |
| Ошибки мерчанта | События `failed` или `error` в цепочке мерчанта | Открывайте журналы мерчанта и настройки платежного провайдера |
| Общий score качества | Сводная оценка по успешности и проблемам | Это сигнал, а не финансовый отчет |

### Команда и операторы

| Блок | Что означает | Что делать |
|---|---|---|
| Админы онлайн | Кто сейчас активен в админке | Полезно для контроля смены |
| Нагрузка операторов | Сколько активных заявок закреплено за оператором | Если один оператор перегружен, перераспределите заявки |
| Текущая страница | Где находится администратор | Помогает понять, кто сейчас работает с заявками |

### Риски

| Риск | Что означает | Действие |
|---|---|---|
| Отклоненные заявки | За период есть заявки со статусом отказа | Сравнить причины отказа и источник трафика |
| Ошибки мерчанта | Платежный провайдер часто возвращает ошибки | Проверить лимиты, ключи, сеть, доступность сервиса |
| Зависшие процессы | Заявки давно не меняли статус в критичных состояниях | Открыть список заявок и проверить каждую спорную заявку |

### Трафик, страны, клиенты и направления

Рабочий стол догружает тяжелую аналитику отдельно, чтобы первый экран открывался быстро.

| Блок | Что показывает | Когда полезно |
|---|---|---|
| График заявок | Созданные, завершенные заявки и новые пользователи | Чтобы увидеть всплески и просадки |
| Трафик | Просмотры, уникальные посетители, пик онлайна | Чтобы связать рекламу и нагрузку |
| Страны | География заявок и прибыль по странам | Для настройки стран, языков и ограничений |
| Популярные направления | Топ направлений по успешным заявкам | Чтобы понять, какие пары развивать |
| Лучшие клиенты | Клиенты с большим числом успешных заявок | Для VIP-условий и поддержки |
| Партнеры | Активные партнеры, партнерские заявки, бонусы | Для контроля партнерской программы |
| Качество партнеров | Завершенные, проблемные заявки и прибыль по партнерскому трафику | Чтобы найти слабые источники трафика |

## Поиск по админке

Глобальный поиск находится в верхней панели и открывается кликом в поле или горячей клавишей `Cmd+K` на Mac и `Ctrl+K` на Windows/Linux. Он нужен, когда пользователь не знает точный раздел или хочет быстро перейти к объекту.

### Что умеет поиск

| Возможность | Что означает | Пример запроса |
|---|---|---|
| Поиск разделов | Находит пункты меню админки | `резервы`, `AML`, `валюты`, `направления` |
| Поиск сущностей | Ищет по backend-источникам, если у пользователя есть права | ID заявки, email пользователя, название валюты |
| Быстрые области | Можно сузить поиск по типу | Заявки, пользователи, валюты, направления, мерчанты, BestChange |
| Префиксы | Быстрый фильтр через короткий код | `o: 12345`, `u: mail@example.com`, `c: USDT`, `d: BTC` |
| Естественные подсказки | Переводит фразу в фильтр страницы | `заявка 12345`, `пользователь test@mail.com`, `резервы USDT` |
| История | Показывает последние запросы администратора | Удобно при повторной проверке одной проблемы |
| Проблемы | Показывает активные anomaly-сигналы | Полезно, если администратор не знает, где поломка |

### Области поиска

| Область | Что искать | Когда выбирать |
|---|---|---|
| Все | Разделы, объекты и подсказки | По умолчанию |
| Разделы | Только пункты админ-меню | Когда нужен путь, а не объект |
| Заявки | Заявки и фильтры заявок | Когда есть номер заявки |
| Пользователи | Пользователи, email, ID | Когда вопрос по клиенту |
| Валюты | Валюты и фильтры списка валют | Когда валюта не видна или нужна настройка |
| Направления | Направления обмена | Когда ищете пару |
| Мерчанты | Мерчанты и автовыплаты | Когда проблема с приемом или выплатой |
| Курсы / BestChange | Источники курсов и BestChange | Когда проблема с курсом |
| Проблемы | Только активные предупреждения | Когда ищете, что сломалось |

### Как правильно искать

| Нужно найти | Как лучше спросить | Если не найдено |
|---|---|---|
| Заявку | `o: 12345` или номер заявки | Откройте "Заявки" и проверьте фильтры/архив |
| Пользователя | `u: email@example.com` | Проверьте точность email и права доступа |
| Валюту | `c: USDT`, `Tether`, `TRC20` | Откройте список валют и фильтры |
| Направление | `d: BTC USDT`, название пары | Проверьте статус валют и направление |
| Раздел | Название раздела простыми словами | Проверьте права роли и модуль |
| Ошибку | Ключевое слово из ошибки | Откройте соответствующий журнал |

### Почему поиск может не найти результат

| Причина | Что сделать |
|---|---|
| Введено меньше 2 символов | Введите более точный запрос |
| Нет прав на раздел или объект | Проверить роль администратора |
| Объект в архиве или удален | Открыть нужный список и включить архив/фильтры |
| Запрос слишком общий | Использовать ID, email, код валюты или префикс |
| Anomaly еще не обновилась | Подождать несколько минут или открыть журнал вручную |

## Аналитика

Раздел "Аналитика" показывает не только прибыль. Он объединяет заявки, направления, валюты, резервы, пользователей, партнеров, трафик и системные показатели.

### Общие фильтры

| Фильтр | Что означает | Пример |
|---|---|---|
| Дата от / до | Период расчета | За сегодня, неделю, месяц |
| Статус | Группа статусов заявок | Успешные, в обработке, отклоненные |
| Устройство | Desktop, mobile, tablet, unknown | Проверка конверсии мобильных клиентов |
| Direction ID | Конкретное направление | Проверить пару BTC -> USDT |
| Partner ID | Конкретный партнер | Спор по партнерскому трафику |
| Referral link ID | Конкретная ссылка | Проверить кампанию |
| User ID | Конкретный клиент | История активного клиента |
| Manager ID | Менеджер/оператор | Контроль работы команды |

### Разделы аналитики

| Раздел | Что показывает | Когда использовать |
|---|---|---|
| Обзор | Сводка заявок, оборота, прибыли, качества | Быстрый отчет владельцу |
| Прибыль обменов | Profit USD, комиссии, финансовая точность | Когда вопрос "сколько заработали" |
| Заявки | Количество, статусы, обработка | Когда нужно понять нагрузку |
| Направления | Лучшие и проблемные пары | Для оптимизации курсов и лимитов |
| Валюты | Оборот и прибыль по валютам | Для выбора популярных валют |
| Резервы | Текущий баланс, поток вход/выход, покрытие | Когда резервов не хватает |
| Партнеры | Партнерские заявки, бонусы, ROI, качество | Для партнерской программы |
| Пользователи | Новые, активные, возвращающиеся клиенты | Для CRM и удержания |
| Трафик | Онлайн, маршруты, устройства, дни, часы, админы | Для рекламы и нагрузки |
| Система | Диагностика фоновых расчетов аналитики | Если цифры устарели |

### Как читать прибыль

Прибыль в аналитике зависит от завершенных заявок, сохраненных записей прибыли и выбранного периода. Если заявка еще в обработке или прибыль не была рассчитана, сумма может не попасть в итог.

| Ситуация | Что проверить |
|---|---|
| Прибыль меньше ожидаемой | Период, статусы, завершение заявок, комиссии направления |
| Прибыль 0 | Есть ли завершенные заявки и profit-записи |
| В отчете нет сегодняшней заявки | Ее статус, дату создания/закрытия и период фильтра |
| Партнерская прибыль не совпадает | Тип начисления партнерки и направление |

## Журнал событий

Журналы отвечают на вопрос "что произошло". Их открывают не для настройки, а для расследования: кто изменил статус, какой сервис вернул ошибку, почему AML остановил заявку, почему не начислился партнерский бонус.

### Какие журналы есть

| Журнал | Что показывает | Когда открывать |
|---|---|---|
| Пользователи | Входы, результат авторизации, IP, устройство, гео, риск | Вопросы доступа и безопасности |
| Мульти-авторизация | Переключение/аудит нескольких админ-аккаунтов | Когда нужно понять, кто работал под кем |
| Активные сессии | Текущие устройства и сессии | Подозрение на чужой доступ |
| Реферальные события | Регистрация, привязка клиента, начисление, отказ, HOLD, сторно | Спор по партнерскому бонусу |
| KYC заявители | Заявки на проверку личности | Клиент не проходит KYC |
| KYC логи | Ответы и статусы KYC-сервисов | Проверка зависла или отказана |
| Статусы заявок | История переходов статуса, курс, сумма, пользователь | Кто и когда перевел заявку |
| AML заявок | Проверка адреса/транзакции, метод, риск, детали | Высокий risk score или блокировка |
| Реквизиты | Выдача, изменение, отключение реквизитов | Клиент спорит по номеру карты/кошелька |
| Flow мерчанта | Стадии приема оплаты, failed/error | Мерчант не выдал реквизиты или платеж не подтянулся |
| Логи шлюзов | API-ответы мерчантов и автовыплат | Диагностика провайдера |

### Уровни событий

| Уровень | Что означает | Реакция |
|---|---|---|
| info | Нормальное информационное событие | Использовать для истории |
| warning | Предупреждение, условие не выполнено | Проверить настройки и контекст |
| error | Ошибка операции | Проверить журнал, заявку и сервис |
| success | Успешное действие, если журнал использует такой результат | Использовать как подтверждение |

### Как расследовать заявку

1. Найдите заявку по ID через поиск или список заявок.
2. Откройте карточку заявки и посмотрите текущий статус.
3. Откройте "Журналы -> Заявки -> Статусы" и найдите ID заявки.
4. Если проблема с оплатой, откройте flow мерчанта и логи платежного шлюза.
5. Если проблема с реквизитами, откройте журнал реквизитов.
6. Если заявка остановлена AML, откройте AML-журнал.
7. Если был KYC, проверьте KYC-заявителей и KYC-логи.
8. Если вопрос по партнерке, откройте реферальные логи по `task_id`.

### Как отвечать клиенту по журналу

| Что видно в журнале | Что можно сказать клиенту |
|---|---|
| Мерчант вернул ошибку | "Платежный сервис не подтвердил операцию, заявка проверяется оператором." |
| AML risk высокий | "Заявка требует дополнительной проверки по правилам безопасности." |
| Реквизит изменен оператором | "Реквизиты были обновлены, проверьте актуальные данные в заявке." |
| KYC ожидает документы | "Для продолжения нужна проверка личности/карты." |
| Автовыплата failed | "Выплата не прошла автоматически, оператор проверит ее вручную." |

Не копируйте клиенту raw-ответы платежных сервисов, IP, внутренние trace ID, реквизиты других клиентов и технические ошибки с приватными данными.

## Частые проблемы

| Проблема | Где смотреть | Что делать |
|---|---|---|
| Рабочий стол показывает пустые резервы | Рабочий стол -> Резервы | Открыть "Резервы", проверить сумму и файл |
| Поиск показывает проблему | Глобальный поиск -> Проблемы | Открыть результат и связанный журнал |
| Аналитика не совпадает с заявками | Аналитика + список заявок | Проверить период, статус, дату закрытия |
| В журнале один IP у всех | Журналы пользователей + настройки прокси | Проверить trusted proxy/real IP |
| Нет свежих событий партнерки | Настройки партнерки | Включить логирование реферальной системы |
| Нет свежей аналитики | CRON, очереди, analytics health | Проверить фоновые процессы |
