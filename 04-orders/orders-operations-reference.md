# Заявки и действия оператора

Заявка — центральный объект обменника. В ней сходятся клиент, направление, сумма, курс, реквизиты, мерчант, AML/KYC, комментарии, чат, статусы, резервы, выплаты и логи.

## Основные статусы заявок

| Статус | Что означает простыми словами | Что обычно делает оператор |
|---|---|---|
| Ожидает оплаты | Клиент создал заявку, но оплата ещё не подтверждена | Ждать оплату или проверить реквизиты |
| Проверка оплаты | Система или оператор проверяет поступление | Проверить мерчант, транзакцию, чек |
| Подтверждение мерчанта | Мерчант ещё не подтвердил платёж | Смотреть историю оплаты и журналы платежных шлюзов |
| Оплачена | Оплата подтверждена | Готовить выплату или переводить дальше |
| В очереди выплаты | Выплата ожидает выполнения | Проверить очередь и лимиты |
| Выплата выполняется | Автовыплата запущена | Ждать результата от провайдера выплат |
| Ошибка автовыплаты | Автовыплата не прошла | Проверить журналы платежных шлюзов и выполнить вручную или повторить |
| В обработке | Оператор работает с заявкой | Следовать регламенту |
| Заморожена | Заявка остановлена для проверки | Проверить причины, AML/KYC, спорные данные |
| Выполнена | Обмен завершён | Обычно действий не нужно |
| Отклонена | Заявка отказана обменником | Проверить причину и комментарий |
| Отменена клиентом | Клиент отменил заявку | Обычно действий не нужно |
| Просрочена | Время оплаты истекло | Проверить, не пришла ли поздняя оплата |
| Недействительная | Заявка признана некорректной | Проверить причины |
| Удалена | Заявка удалена или скрыта | Использовать только по регламенту |

## Статусы оплаты внутри заявки

| Статус оплаты | Что означает | Что проверить |
|---|---|---|
| Создано | Платёжный процесс создан | Есть ли реквизиты и сумма |
| Ожидает транзакцию | Система ждёт поступление | Мерчант, сеть перевода, чек клиента |
| Регистрация | Платёж регистрируется | Ответ провайдера |
| Зарегистрировано | Платёж найден у провайдера | Ждать подтверждения или AML |
| AML-проверка | Проверяется риск транзакции | AML журналы |
| Требуется AML/KYC | Нужна проверка | Валюта, направление, KYC/AML правила |
| Транзакция найдена | Платёж обнаружен | Сумма, валюта, подтверждения |
| Ожидает подтверждений | Нужны подтверждения сети | Количество подтверждений |
| Подтверждения получены | Транзакция подтверждена | Переводить к выплате |
| Успешная оплата | Платёж принят | Продолжить обработку |
| Платёж не прошёл | Ошибка транзакции | Причина в платежном шлюзе или сети |
| Недостаточная сумма | Клиент отправил меньше | Решить: доплата, пересчёт, возврат, отклонение |
| Неверная валюта | Отправлена не та валюта | Нужна ручная проверка и связь с клиентом |

## Список заявок

| Поле / действие | Что означает | Как использовать | Рекомендация |
|---|---|---|---|
| ID / публичный ID | Номер заявки | Ищите по нему в первую очередь | Просите у клиента номер заявки |
| Публичный номер заявки | Защищённый идентификатор публичной заявки | Используется в ссылках на заявку | Не публикуйте внутренние связи |
| Статус | Текущее состояние | Фильтруйте активные, оплаченные, спорные | Не меняйте статус без проверки |
| Валюта “Отдаю” | Что клиент отправляет | Проверяйте при платеже | Сторона важна |
| Валюта “Получаю” | Что клиент получает | Проверяйте при выплате | Сторона важна |
| Сумма отдачи | Сколько клиент должен оплатить | Сверяйте с мерчантом и чеком | Может пересчитаться |
| Сумма получения | Сколько клиент должен получить | Сверяйте с резервом и выплатой | Учитывайте комиссии |
| Клиент | Пользователь заявки | Открывайте профиль при повторных проблемах | Не раскрывайте данные без причины |
| Email / телефон / Telegram | Контакты клиента | Используйте для связи | Не копируйте в публичные документы |
| Дата создания | Когда создана заявка | Сравнивайте с временем оплаты | Важен часовой пояс |
| Оператор | Кто работает с заявкой | Фильтруйте по сотруднику | Помогает распределять работу |
| Режим живого обновления | Обновление списка в реальном времени | Используйте для рабочей смены | Может зависеть от настроек уведомлений и очередей |
| Экспорт | Выгрузка заявок | Для отчётов | Доступ только доверенным |

## Фильтры заявок

| Фильтр | Что означает | Как использовать |
|---|---|---|
| Поиск | ID, email, реквизит, сумма, клиент | Сначала ищите по ID, затем по данным |
| Статус | Ограничить список по состоянию | Для рабочих очередей |
| Период | Дата создания или изменения | Для поиска старых проблем |
| Направление | Пара обмена | Например `BTC - USDT` |
| Валюта | Одна из сторон заявки | Если не знаете точную пару |
| Оператор | Кто назначен | Для контроля смены |
| Мерчант | Через кого была оплата | Для проблем платежного шлюза |
| AML/KYC | Проверки и блокировки | Для compliance-задач |
| Архив | Старые заявки | Если не находится в активном списке |

## Карточка заявки

| Блок | Что показывает | Когда использовать |
|---|---|---|
| Основная информация | ID, статус, направление, суммы, курс | Всегда при открытии заявки |
| Клиент | Данные пользователя и контакты | Для связи и проверки истории |
| Реквизиты клиента | Счета отправки и получения | При проверке оплаты и выплаты |
| Поля “Отдаю” | Доп. поля входящей стороны | Если не хватает банка, карты, документа |
| Поля “Получаю” | Доп. поля выплаты | Если автовыплаты требует больше данных |
| Курс | Курс заявки и тип курса | При споре по сумме |
| Мерчант | Провайдер приёма оплаты | Если реквизиты не выданы или оплата не пришла |
| Автовыплата | Провайдер выплаты | Если выплата зависла |
| AML | Результаты проверки риска | Если заявка заморожена или требует проверки |
| KYC | Верификация клиента | Если нужна личность или документы |
| Комментарии оператора | Внутренние заметки | Для смены операторов |
| Комментарии клиенту | Сообщения, видимые клиенту | Для объяснения действий |
| Фото / вложения | Чеки и файлы | При споре оплаты |
| История статусов | Кто и когда менял статус | Для аудита |
| История операторов | Кто брал заявку | Для контроля работы |
| Чат | Переписка по заявке | Для уточнения реквизитов и чеков |

## Действия оператора

| Действие | Что делает | Когда использовать | Предупреждение |
|---|---|---|---|
| Принять в работу | Назначить заявку оператору | Когда оператор начинает обработку | Не берите заявку без намерения обработать |
| Освободить заявку | Снять назначение | Если оператор не может продолжить | Комментарий помогает смене |
| Заблокировать подключение операторов | Не давать другим присоединяться | При спорной заявке или личной ответственности | Не злоупотребляйте |
| Разблокировать подключение | Разрешить другим работать | Когда спор снят | Проверьте историю |
| Проверить оплату | Запустить проверку поступления | Если клиент нажал “Я оплатил” или есть чек | Сначала проверьте реквизиты и сумму |
| Остановить автопроверку оплаты | Прекратить автоматическую проверку | Если процесс ошибочный или мешает | Используйте осторожно |
| Запросить реквизиты оплаты | Выдать клиенту реквизиты вручную | Если мерчант не выдал или нужен ручной счёт | Не вставляйте чужие реквизиты |
| Пересчитать заявку | Обновить сумму по текущему курсу | Если включён пересчёт и оплата просрочена | Объясните клиенту изменение |
| Подтвердить оплату | Перевести заявку как оплаченную | Только после фактической проверки | Ошибка влияет на деньги |
| Запустить выплату | Отправить средства клиенту | После подтверждения оплаты и проверок | Проверьте сеть и адрес |
| Повторить выплату | Повторить после ошибки | Только если первая выплата точно не прошла | Риск двойной выплаты |
| Отклонить заявку | Отказать в обмене | При неоплате, scam, неверных данных | Укажите причину |
| Заморозить | Остановить обработку | При AML/KYC, споре, подозрении | Нужен комментарий |
| Завершить | Закрыть как выполненную | После успешной выплаты | Проверьте номер транзакции |
| Добавить комментарий | Оставить внутреннюю заметку | Для передачи смены | Не пишите грубо и не вставляйте секреты |
| Сообщение клиенту | Написать видимое сообщение | Для запроса чека или данных | Пишите простым языком |
| Прикрепить фото | Добавить чек или доказательство | При ручной проверке | Не добавляйте лишние персональные данные |

## Пересчёт заявки

| Поле / настройка | Что означает | Как использовать | Рекомендация |
|---|---|---|---|
| Старый курс | Курс при создании заявки | Сравните с новым | Показывает причину изменения |
| Новый курс | Актуальный курс | Применяйте по правилам направления | Не пересчитывайте без регламента |
| Старая сумма | Сумма до пересчёта | Покажите в споре | Важна для объяснения |
| Новая сумма | Сумма после пересчёта | Проверить перед сохранением | Ошибка влияет на выплату |
| Причина | Почему пересчитали | Укажите понятно | Нужно для логов |
| Audit | История пересчётов | Смотреть при спорах | Показывает кто и когда пересчитал |

## Комментарии и чат

| Поле / действие | Что означает | Как использовать | Рекомендация |
|---|---|---|---|
| Внутренний комментарий | Видят только админы | Для смены, проверки, предупреждений | Не виден клиенту |
| Комментарий клиенту | Видит клиент | Для запроса чека, уточнения данных | Пишите вежливо и конкретно |
| Удалить комментарий | Убрать ошибочную запись | Только если это разрешено регламентом | Лучше не удалять важную историю |
| Чат заявки | Переписка в реальном времени | Для поддержки клиента | Проверьте уведомления |
| Закрыть чат | Запретить новые сообщения | После завершения спора | Клиент может потерять канал связи |
| Разрешить файлы | Клиент может прислать чек | Включайте при необходимости | Ограничьте типы файлов |
| Запросить поле | Попросить клиента заполнить недостающие данные | Для адреса, банка, ФИО | Не просите лишние данные |

## Лимиты операторов

| Поле / настройка | Что означает | Как заполнять | Рекомендация |
|---|---|---|---|
| Оператор | Сотрудник | Выберите пользователя | Проверяйте роль |
| Группа лимитов | Категория ответственности | Например junior, senior | Удобно управлять командой |
| Название лимита | Имя правила | Пишите по смыслу | Например “до 500 USDT” |
| От суммы | Нижняя граница | Укажите сумму | Диапазоны не должны конфликтовать |
| До суммы | Верхняя граница | Укажите сумму | Крупные заявки отдавайте старшим |
| Статус | Включено ли правило | Включайте после проверки | Ошибка может скрыть заявки от оператора |

| Поле / настройка | Что означает | Как использовать | Рекомендация |

## Типовые ошибки по заявкам

| Ситуация | Причина | Что сделать |
|---|---|---|
| Клиент оплатил, но заявка ждёт оплату | Уведомление от провайдера не пришло, перевод без подтверждений или сумма не совпала | Проверить историю оплаты, сеть перевода и журналы платежных шлюзов |
| Клиент отправил меньше | Неполная оплата | Решить по регламенту: доплата, пересчёт, возврат, отклонение |
| Автовыплата не ушла | Статус, лимит, сеть, ошибка шлюза, очередь | Проверить настройки автовыплаты и журналы |
| Заявка не пересчитывается | Статус не разрешён, пересчёт выключен, порог не достигнут | Проверить настройки направления |
| Оператор не видит действие | Нет права или статус не допускает действие | Проверить роль и этапы обработки |
| Клиент не видит заявку в ЛК | Статус не включён в отображаемые | Проверить настройки обмена |
| Чек не открывается | Ограничен доступ по IP/владельцу | Проверить режим доступа к чеку |
| Заявка попала в архив | Автоархив по статусу и возрасту | Ищите в архиве |

## Безопасная обработка спорной заявки

1. Найдите заявку по ID.
2. Проверьте статус, сумму, курс и направление.
3. Проверьте реквизиты, которые видел клиент.
4. Проверьте платежи: историю оплаты, журналы платежных шлюзов, просмотр транзакции в сети и чек.
5. Проверьте AML/KYC, если заявка заморожена или рискованная.
6. Добавьте внутренний комментарий с фактами.
7. Только после этого меняйте статус или делайте выплату.
8. Клиенту пишите коротко: что проверяется, что нужно от него, какой следующий шаг.
