# Email-уведомления

## Для чего нужен раздел

Email-уведомления отвечают за письма клиентам и администраторам: изменение статуса заявки, успешный вход, подтверждение e-mail, верификацию, новые заявки, отзывы и служебные события.

## Когда использовать

Открывайте раздел почты, когда письма не приходят, меняется почтовый сервис, нужно включить или отключить определенные уведомления, изменить адрес отправителя или отправить тестовое письмо.

## Основные возможности

| Возможность | Что означает простыми словами | Когда использовать |
| --- | --- | --- |
| Почтовый драйвер | Способ отправки писем: Resend или SMTP | При подключении почты |
| Адрес отправителя | От кого приходят письма | При настройке бренда и доставляемости |
| Адрес поддержки | Куда клиент может ответить | Когда меняется поддержка |
| Адреса админов | Кто получает служебные письма | Для новых заявок и проверок |
| Общий тумблер уведомлений | Полностью включить или выключить уведомления | Для аварийной остановки |
| Тумблер писем | Включить или выключить именно email | Когда Telegram работает, а письма нужно остановить |
| Клиентские события | Письма клиентам | Чтобы управлять опытом клиента |
| Админские события | Письма сотрудникам | Чтобы не пропустить заявку или проверку |
| Тестовое письмо | Проверка доставки | После изменения настроек |

## Поля и настройки

| Поле / настройка | Что означает | Как правильно заполнять | Важно знать |
| --- | --- | --- | --- |
| Почтовая служба | Resend или SMTP | Выберите используемый сервис | Неверный драйвер ломает отправку |
| Resend API key | Ключ Resend | Заполняйте только в админке | Нельзя добавлять в документацию |
| SMTP host | Сервер отправки | Укажите host провайдера | Ошибка host даст сбой отправки |
| SMTP port | Порт сервера | Используйте порт провайдера | Часто это 465, 587 или 25 |
| SMTP username | Логин | Заполняйте, если требует провайдер | Для localhost может быть не нужен |
| SMTP password | Пароль | Заполняйте только в админке | Секретное поле |
| Шифрование | SSL или TLS | Выберите по инструкции провайдера | Неверное шифрование блокирует соединение |
| E-mail отправителя | Адрес, откуда идут письма | Используйте доменный адрес проекта | Некоторые провайдеры требуют подтвержденный адрес |
| Имя отправителя | Имя в письме | Укажите бренд обменника | Клиент видит это имя в почте |
| E-mail поддержки | Адрес для ответов клиентов | Укажите рабочий адрес поддержки | Должен регулярно проверяться |
| E-mail администраторов | Получатели служебных писем | Несколько адресов разделяйте запятой | Неверный адрес потеряет уведомления |

## Как настроить

1. Откройте настройки почтовой службы.
2. Выберите Resend или SMTP.
3. Заполните данные подключения.
4. Укажите e-mail отправителя и имя отправителя.
5. Укажите адрес поддержки и адреса администраторов.
6. Включите общий тумблер уведомлений и тумблер email.
7. Выберите, какие события отправлять клиентам и администраторам.
8. Отправьте тестовое письмо.
9. Если письмо не пришло, проверьте лог и настройки провайдера.

## Типовые ошибки

| Ошибка / ситуация | Причина | Что сделать |
| --- | --- | --- |
| Письмо не отправляется | Неверные SMTP или Resend настройки | Проверить драйвер, host, port, ключ и пароль |
| Письмо уходит в спам | Неверный отправитель или домен не подтвержден | Настроить доменную почту у провайдера |
| Админ не получает новые заявки | Адрес не указан или уведомление выключено | Проверить адреса администраторов и тумблеры |
| Клиент не получает статус заявки | Выключено событие изменения статуса | Включить клиентское уведомление |
| Тестовое письмо не приходит | Ошибка подключения или провайдер заблокировал отправку | Проверить лог и настройки сервиса |

## Важные предупреждения

Не добавляйте в документацию SMTP-пароль, Resend API key, реальные адреса клиентов и содержимое писем конкретных заявок.

При отключении общего тумблера уведомлений письма и другие уведомления могут перестать уходить всем получателям.
