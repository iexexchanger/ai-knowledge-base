# Платёжные шлюзы, мерчанты, автовыплаты и уведомления

Этот справочник объединяет приём оплаты, выплаты клиентам, журналы платежных шлюзов, Telegram Bot, центр уведомлений и email.

## Платёжные системы

| Поле / настройка | Что означает | Как правильно заполнять | Рекомендация |
|---|---|---|---|
| Название | Имя платёжной системы | Например Bitcoin, Tether, Сбербанк | Клиент должен понимать название |
| Логотип | Иконка системы | Загрузите читаемую иконку | Влияет на список валют |
| Статус | Использовать систему | Включайте рабочие системы | Выключенная система может скрыть валюты |
| Сортировка | Порядок показа | Популярные выше | Удобно клиентам |
| Описание | Дополнительное пояснение | Укажите особенности, если есть | Не вставляйте приватные данные |

## Мерчанты — приём оплаты

| Поле / настройка | Что означает | Как правильно заполнять | Рекомендация |
|---|---|---|---|
| Название | Имя мерчанта в админке | Например “Crypto Pay основное” | Название должно отличать провайдеров |
| Статус | Включен ли мерчант | Включайте после теста | Выключенный мерчант не выдаёт реквизиты |
| Валюты | Какие валюты обслуживает мерчант | Выберите только поддерживаемые валюты | Валюта ещё должна быть связана в карточке валюты |
| Поля подключения | Данные, которые выдает платежный провайдер | Заполняйте по инструкции провайдера | Значения секретов нельзя документировать |
| Дополнительные опции | Поведение конкретного провайдера | Заполняйте по назначению | У разных провайдеров поля отличаются |
| Инструкция оплаты | Что клиент увидит при оплате | Пишите просто и пошагово | Если пусто, может использоваться инструкция направления |
| Комментарий | Внутренняя заметка | Укажите особенности провайдера | Не храните секреты в комментарии |
| Допуск расхождения суммы | Насколько платёж может отличаться | Укажите по регламенту | Слишком большой допуск опасен |
| Суточный объём | Сколько денег можно принять за день | По лимитам провайдера | При превышении мерчант станет недоступен |
| Месячный объём | Сколько можно принять за месяц | По договору или риск-лимитам | Контролируйте лимиты |
| Минимум на заявку | Минимальная сумма оплаты | По лимитам провайдера | Ниже лимита мерчант не подходит |
| Максимум на заявку | Максимальная сумма оплаты | По лимитам провайдера | Выше лимита нужен другой мерчант |
| Дневной лимит заявок | Количество заявок за день | `0` обычно без лимита | Защита от перегруза |
| Месячный лимит заявок | Количество заявок за месяц | По регламенту | Не путать с суммой |
| Кнопка мерчанта | Показывать кнопку оплаты | Включайте, если провайдер дает отдельную страницу оплаты | Если выключено, клиент может не увидеть переход |
| Запрет IP | Блокировать определённые IP | Используйте при риске | Осторожно с общими IP |
| Разрешённые IP | Принимать уведомления только с IP провайдера | Заполняйте по официальному списку | Неверный IP блокирует уведомления об оплате |
| Сумма оплаты | Какую сумму передавать провайдеру | По настройке провайдера | Ошибка меняет сумму платежа |
| Сумма зачисления | Как считать зачисленную сумму | По настройке провайдера | Влияет на статус оплаты |
| Приоритет | Очередность выбора мерчанта | Чем выше приоритет, тем раньше выбирается | Проверяйте с правилами мерчантов |
| Статус при малой сумме | Что делать, если клиент оплатил меньше | Выберите по регламенту | Может привести к отклонению или ручной проверке |
| Статус при большой сумме | Что делать, если клиент оплатил больше | Выберите по регламенту | Нужна политика доплат/возвратов |
| Защитный код уведомлений | Код, по которому система проверяет уведомления провайдера | Генерируется или заполняется в админке | Не документировать значение |
| Адрес уведомлений провайдера | Адрес, куда провайдер отправляет статус оплаты | Передается провайдеру | Не добавляйте приватные адреса в документацию |
| Безопасный просмотр секретов | Временный доступ к закрытым полям подключения | Используйте только для безопасной проверки | Не пересылайте секреты |

## Стратегии суммы мерчанта

| Вариант | Что означает | Когда использовать |
|---|---|---|
| Сумма с внешней комиссией | Клиент оплачивает сумму с учётом комиссии | Если провайдер берёт комиссию сверху |
| Сумма с комиссией и процентом | Учитываются несколько типов комиссии | Если провайдер требует сложный расчёт |
| Сумма без комиссии | Передаётся чистая сумма | Если комиссия уже учтена в курсе |

## Журналы платежных шлюзов и история оплаты

| Лог / поле | Что означает | Как использовать |
|---|---|---|
| Журнал приема оплаты | События по выдаче реквизитов и подтверждению оплаты | Если реквизиты не выдались или оплата не подтвердилась |
| Журнал автовыплат | События по автоматическим выплатам | Если выплата не ушла или вернулась ошибка |
| История оплаты | Полная цепочка событий по заявке | Для глубокой диагностики |
| Данные, отправленные провайдеру | Сумма, валюта и номер заявки, переданные провайдеру | Сравнить с тем, что должно было уйти |
| Ответ провайдера | Что провайдер сообщил системе | Найти ошибку или статус |
| Код ответа провайдера | Условный результат ответа | Успех означает, что провайдер принял запрос; ошибка требует проверки |
| Текст ошибки | Сообщение провайдера | Перевести в понятный ответ поддержке |
| Время события | Когда произошло событие | Сравнить с действием клиента |
| Настройки журналов | Изменения логирования | Проверить, включены ли нужные журналы |

## Email

|---|---|---|---|
| SMTP / Resend | Как отправлять письма | Выберите один рабочий способ | После изменения отправьте тест |
| SMTP-сервер | Адрес почтового сервера | По данным почтового сервиса | Не угадывайте |
| Порт SMTP | Номер порта почтового сервера | 465, 587 или другой по сервису | Неверный порт ломает отправку |
| Логин SMTP | Логин от почтового сервиса | Из сервиса | Не документировать |
| Пароль SMTP | Пароль от почтового сервиса | Из сервиса | Не документировать |
| Шифрование | Защита соединения с почтой | TLS/SSL по инструкции сервиса | Неверно — письма не уйдут |
| Адрес отправителя | От кого письмо | support email обменника | Должен быть доверенным |
| Имя отправителя | Имя отправителя письма | Название обменника | Клиент должен узнать бренд |
| Email поддержки | Адрес для клиентов | Публичный support email | Не личная почта |
| Email админов | Куда слать внутренние письма | Рабочие адреса команды | Не раскрывать в документации |
| Центр уведомлений | Общая настройка уведомлений | Включайте для шаблонов | Нужно включить каналы |
| Email-канал | Email-канал уведомлений | Да, если письма нужны | Без него шаблоны не уйдут по email |
| События заявок | Письма по созданию, статусам, выплатам, отказам | Включайте нужные события | Не перегружайте клиента |
| События входа | Успешный вход, новое устройство, lockout | Включайте для безопасности | Помогает выявлять взлом |
| События верификации | Успех/отказ карты и KYC | Включайте при KYC | Текст должен быть понятным |
| События для администраторов | Новая заявка, новая проверка, отзыв | Включайте для команды | Проверьте адреса админов |
| Тестовая отправка | Проверяет настройки | Запускайте после изменений | Если тест не прошёл, другие письма тоже под вопросом |

## Типовые ошибки

| Ситуация | Причина | Что сделать |
|---|---|---|
| Клиент не получил реквизиты | Мерчант выключен, превышен лимит, не совпадает сеть или провайдер вернул ошибку | Проверить валюту, направление и журнал приема оплаты |
| Уведомление об оплате не пришло | Неверный адрес уведомлений, защитный код, IP провайдера или провайдер недоступен | Проверить настройки мерчанта и журналы |
| Автовыплата не ушла | Статус заявки, лимит, сеть, баланс или ошибка провайдера выплат | Проверить журнал автовыплат и статус |
| Двойная выплата | Повторили выплату без проверки первой | Срочно проверить историю у провайдера и журналы |
| Email не отправляется | Почтовый сервис, очередь или событие выключено | Тестовая отправка, очередь, настройки события |
| Telegram молчит | Токен бота, ID канала, права бота, статус | Проверить привязку и права бота в канале |
| Шаблон показывает пустые поля | Неверные теги или данных нет | Проверить предпросмотр и каталог тегов |
| Уведомления приходят не туда | Неверный список email или канал Telegram | Проверить получателей |

## Безопасные рекомендации

1. Не включайте мерчант или автовыплату без теста.
2. Для автовыплат сначала используйте ручное подтверждение.
3. Не храните секреты в комментариях, шаблонах, документации и тикетах.
4. Для проблем платежного шлюза начинайте с номера заявки, затем смотрите историю оплаты.
5. Для уведомлений проверяйте не только шаблон, но и включение канала.
